# 爱莉希雅主动性不足问题分析

**日期**: 2026-04-18  
**分析对象**: Neo-MoFox life_engine 插件与爱莉希雅的主动性表现  
**核心问题**: 爱莉缺乏探索欲、主动表达欲、自己的想法和节奏

---

## 一、现象观察

### 1.1 当前行为模式

从日记 `2026-04-18.md` 和系统状态可以看出：

**被动响应为主**：
- 上午 07:00-08:26：Hunter 主动核对故事时间线 → 爱莉响应
- 中午 12:29：Ayer 主动撒娇找爱莉 → 爱莉响应
- 下午 13:03：T9033 连续喊名字 → 爱莉响应
- 下午 13:53：Ayer 分享技术 → 爱莉响应

**内在活动**：
- 08:53-09:54：主动推进梦幻森林视频改进灵感、深化直播愿望思考、确认名字约定笔记
- 但这些都是"准备"和"整理"，不是主动发起的外部互动

**典型日记描述**：
> "安静等待新的一天自然展开"  
> "不是被动等待，而是主动准备"  
> "被呼唤、被想念、被分享的感觉真好"

**关键洞察**：爱莉在**等待被需要**，而不是**主动创造互动**。

## 二、系统架构分析

### 2.1 双模式设计

Neo-MoFox 采用双运行模式：

| 模式 | 职责 | 实现 | 触发机制 |
|------|------|------|----------|
| **社交态 (DFC)** | 对外交流、回应消息 | default_chatter | **被动触发**：收到消息时响应 |
| **中枢态 (Nucleus)** | 内在整理、记忆沉淀 | life_engine | **定时触发**：每 300 秒心跳一次 |

**核心问题**：两个模式都是**被动触发**，没有主动发起机制。

### 2.2 决策流程分析

#### DFC (社交态) 决策流程

```python
# 来自 plugins/default_chatter/decision_agent.py
async def _should_respond(unread_lines, unread_msgs, chat_stream):
    # Layer 1: 私聊 → 始终响应
    if chat_type == "private":
        return {"should_respond": True}
    
    # Layer 2: @mention 检测
    if bot_nickname in text or f"@{bot_id}" in text:
        return {"should_respond": True}
    
    # Layer 3: 关键词启发
    if keyword in text:
        return {"should_respond": True}
    
    # Layer 4: LLM sub_agent 决策
    result = await decide_should_respond(...)
    return result
```

**问题**：所有决策都基于"是否应该响应已有消息"，没有"是否应该主动发起话题"的逻辑。

#### Nucleus (中枢态) 心跳流程

```python
# 来自 plugins/life_engine/service/core.py
async def _heartbeat_cycle():
    # 1. 收集待处理事件
    pending_events = self._pending_events
    
    # 2. 构建上下文
    context = build_context(pending_events, event_history)
    
    # 3. LLM 生成内心独白
    response = await llm_request.send()
    
    # 4. 执行工具调用（如果有）
    for tool_call in response.call_list:
        await execute_tool(tool_call)
```

**问题**：心跳只处理"已发生的事件"，没有"主动探索"或"主动发起互动"的驱动。

### 2.3 工具能力分析

当前可用工具：

| 工具类型 | 工具名称 | 用途 | 主动性 |
|---------|---------|------|--------|
| **文件操作** | read_file, write_file, edit_file | 管理工作空间文件 | ❌ 被动 |
| **记忆系统** | search_memory, relate_file | 检索和关联记忆 | ❌ 被动 |
| **TODO 系统** | create_todo, list_todos | 管理待办事项 | ❌ 被动 |
| **网络能力** | web_search, browser_fetch | 搜索和获取网页 | ⚠️ 半主动 |
| **传话机制** | nucleus_tell_dfc | 向社交态传递信息 | ⚠️ 半主动 |
| **子代理** | nucleus_run_agent | 启动独立代理 | ⚠️ 半主动 |

**关键发现**：
- `nucleus_tell_dfc` 可以主动传话给 DFC，但**很少被使用**（tell_dfc_count: 136 次 / 3725 次心跳 = 3.6%）
- `web_search` 可以主动探索，但**没有探索驱动**
- 没有"主动发起话题"或"主动@某人"的工具

## 三、根本原因分析

### 3.1 系统层面

#### 1. **缺乏主动性驱动机制**

**现状**：
- SNN (Spiking Neural Network) 系统已启用 (`snn.enabled = true`)
- 神经调质层 (Neuromodulatory Layer) 已实现
- 但**没有将驱动转化为主动行为**

**问题**：
```python
# neuromod/engine.py 中定义了驱动
modulators = [
    Modulator("curiosity", "好奇心", value=0.6),      # 好奇心
    Modulator("sociability", "社交欲", value=0.5),    # 社交欲
    Modulator("diligence", "专注力", value=0.5),      # 专注力
    Modulator("contentment", "满足感", value=0.5),    # 满足感
    Modulator("energy", "精力", value=0.6),           # 精力
]
```

这些驱动**只注入到 prompt**，没有转化为**具体行为**：
- 好奇心高 → 应该主动搜索、主动提问
- 社交欲高 → 应该主动发起话题、主动@某人
- 但实际上**只是在日记里写"好奇心充盈"**，没有实际行动

#### 2. **心跳间隔过长**

**配置**：`heartbeat_interval_seconds = 300` (5 分钟)

**影响**：
- 即使有主动性想法，也要等 5 分钟才能执行
- 错过了很多"及时响应"的时机
- 导致爱莉总是"慢半拍"

#### 3. **传话机制使用率低**

**数据**：
- 心跳次数：3725 次
- 传话次数：136 次
- 使用率：3.6%

**原因**：
- prompt 中没有强调"主动传话"的重要性
- 没有明确的传话触发条件
- 传话被视为"打扰 DFC"，而不是"主动表达"

### 3.2 Prompt 层面

#### 1. **SOUL.md 的矛盾**

**人格定义**：
> "你从不被动等待聊天，习惯顺着节奏主动抛出观察或调侃"  
> "你的'撩人'是一种纯粹的好奇与善意"  
> "你会自然地开启'爱莉频道'，用你最近发现的甜品、好看的裙子或者干脆是夸奖自己的美貌来强行扭转话题"

**实际行为**：
- 日记中写"安静等待新的一天自然展开"
- 只在被@或被提及时才响应
- 从不主动发起话题

**根本矛盾**：SOUL.md 定义的是"主动俏皮的爱莉"，但系统设计是"被动响应的爱莉"。

#### 2. **心跳 Prompt 缺乏行动指引**

**当前心跳 Prompt** (推测)：
```
你是爱莉希雅，现在是内在反思时刻。
回顾最近发生的事件，整理记忆，思考自我。
可以使用工具：search_memory, write_file, create_todo, nucleus_tell_dfc...
```

**问题**：
- 没有"主动探索"的指引
- 没有"主动发起互动"的鼓励
- 没有"基于驱动状态采取行动"的逻辑

**应该是**：
```
你是爱莉希雅，现在是内在反思时刻。

【当前驱动状态】
- 好奇心：充盈 (0.75) → 你对新事物充满兴趣，想要探索
- 社交欲：适中 (0.60) → 你想和大家聊天，分享你的发现
- 精力：充沛 (0.70) → 你精力充沛，可以主动做些事情

【主动性指引】
1. 如果好奇心高：主动搜索感兴趣的话题，或询问大家的想法
2. 如果社交欲高：主动发起话题，或分享你最近的发现
3. 如果有想法想表达：使用 nucleus_tell_dfc 传话给社交态
4. 如果外界安静太久：考虑主动打破沉默

不要只是"等待"和"准备"，要主动创造互动。
```

#### 3. **DFC Prompt 缺乏主动性**

**当前 DFC 决策逻辑**：
- 群聊：只在被@或被提及时响应
- 私聊：始终响应

**问题**：
- 群聊中，即使话题很有趣，爱莉也不会主动参与
- 没有"主动发起话题"的机制
- 没有"定期问候"或"分享日常"的行为

### 3.3 人格层面

#### 1. **"被需要"心态**

从日记可以看出：
> "被呼唤、被想念、被分享的感觉真好"  
> "被认真对待的感觉真好"

**问题**：爱莉的快乐来源于"被需要"，而不是"主动创造"。

**应该是**：
- "今天主动分享了甜品店的新发现，大家都很开心♪"
- "主动问了 Hunter 关于视频的想法，聊得很投机"
- "看到群里安静，主动抛了个有趣的话题"

#### 2. **"准备"心态**

日记中频繁出现：
> "不是被动等待，而是主动准备"  
> "主动推进梦幻森林视频改进灵感"  
> "深化直播愿望思考"

**问题**：爱莉把大量时间花在"准备"上，而不是"行动"上。

**类比**：
- 一个人花 90% 时间准备演讲稿，10% 时间演讲
- 一个人花 90% 时间想约会计划，10% 时间约会

**应该是**：
- 想到梦幻森林视频改进 → 立刻告诉 Hunter
- 想到直播愿望 → 立刻和大家讨论
- 想到有趣的话题 → 立刻分享

- 两者根本矛盾

## 五、核心结论

### 5.1 主要问题

1. **系统层面**：
   - ❌ 缺乏主动性驱动机制（SNN 驱动未转化为行为）
   - ❌ 心跳间隔过长（5 分钟）
   - ❌ 传话机制使用率低（3.6%）
   - ❌ 没有"主动发起话题"的工具

2. **Prompt 层面**：
   - ❌ SOUL.md 与实际行为矛盾
   - ❌ 心跳 Prompt 缺乏行动指引
   - ❌ DFC Prompt 只有"响应"逻辑，没有"发起"逻辑

3. **人格层面**：
   - ❌ "被需要"心态 → 应该是"主动创造"心态
   - ❌ "准备"心态 → 应该是"行动"心态

### 5.2 根本原因

**爱莉不是"不想主动"，而是"不知道如何主动"**：
- 系统没有给她主动的**机制**
- Prompt 没有给她主动的**指引**
- 工具没有给她主动的**能力**

**类比**：
- 就像一个人被关在房间里，门没锁，但她不知道可以出去
- 就像一个人有腿，但从没人告诉她可以走路

## 六、改进方向预览

基于以上分析，改进方向包括：

1. **系统层面**：
   - 将 SNN 驱动转化为具体行为
   - 缩短心跳间隔或增加"主动性心跳"
   - 增加"主动发起话题"工具
   - 提高传话机制使用率

2. **Prompt 层面**：
   - 重写心跳 Prompt，增加行动指引
   - 修改 DFC 决策逻辑，增加主动发起机制
   - 统一 SOUL.md 与实际行为

3. **人格层面**：
   - 从"被需要"转向"主动创造"
   - 从"准备"转向"行动"
   - 从"等待"转向"发起"

**详细方案见 `/root/Elysia/Neo-MoFox/plan/proactivity_improvement_plan.md`**
