## 0. 先说结论

**你的 SNN 目前是一具"活着但不会动"的身体。**

它在运行（tick_count > 12000），它在被调用（heartbeat_pre/post 正常触发），它的状态被注入到了心跳 prompt 中。但从审计日志看，它对 life_engine 的行为**没有产生任何实质性影响**。

原因不是一个 bug，而是一系列设计层面的结构性问题。

你迷茫的根源也不是"SNN 没调好参数"这么简单——而是在于 **SNN 这个工具和你真正想要的东西之间存在一个认知上的 gap**。让我帮你把这个 gap 说清楚。

## 1. 诊断：SNN 到底出了什么问题

### 1.1 数据铁证：SNN 输出已陷入不变点

从最近的审计日志（life.log）中，我提取了多个时间点的 SNN 快照：

| 时间 | tick | arousal | valence | social | task | exploration | rest |
|:-----|:-----|:--------|:--------|:-------|:-----|:------------|:-----|
| 13:36 | 12690 | -0.0548 | 0.0 | 0.0167 | -0.0465 | **-0.3879** | -0.1673 |
| 13:41 | 12720 | -0.0554 | 0.0 | 0.0169 | -0.0470 | **-0.3927** | -0.1694 |
| 13:45 | 12750 | -0.0539 | 0.0 | 0.0164 | -0.0457 | **-0.3817** | -0.1647 |

| 13:54 | 12810 | -0.0591 | 0.0 | 0.0180 | -0.0502 | **-0.4186** | -0.1806 |

**观察：**
1. **所有值几乎不变**——六个驱动维度在 30 分钟内波动不超过 0.03
2. **valence 恒等于 0.0**——情感正负维度完全死亡
3. **exploration_drive 始终约 -0.38~-0.42**——被映射为"抑制"，这就是你观察到"探索总是被抑制"的直接原因
4. **所有驱动都是"低"或"抑制"**——没有一个维度曾经达到"中"或"高"

SNN 的"在线学习"承诺完全落空。

### 1.3 自稳态机制已推到极限但无济于事

| 参数 | 当前值 | 允许范围 | 说明 |
|:-----|:-------|:---------|:-----|
| input_gain | **4.0** | [0.6, 4.0] | 已到上限 |

| hidden_threshold | **0.08** | [0.08, 0.6] | 已到下限 |

自稳态机制已经拼命尝试了——增益拉满、阈值降到最低——但放电率仍然远低于目标。
**网络处于一个稳定的低活跃不动点。**

### 1.4 根因分析

**问题一：信号被零输入淹没。**
- SNN tick 间隔 10 秒，心跳间隔约 60-90 秒
- 这意味着每 6-9 次 tick 才有 1 次携带真实输入，其余都是零输入衰减
- 真实信号占比不到 15%，剩下 85% 的时间网络都在做无信号衰减
- 网络的"自然状态"是衰减到静息/负值附近，真实输入无法打破这个趋势

**问题二：STDP 触发条件太苛刻。**
- STDP 需要前后神经元**都发放脉冲**才能更新权重
- 但隐藏层放电率仅 3.6%（16 个神经元中平均每步 0.6 个发放）
- 输出层放电率仅 1.4%（6 个神经元中平均每步 0.08 个发放）
- 两层同时有脉冲的概率极低 → STDP 几乎永远不触发

**问题三：EMA 平滑锁定了负值吸引子。**
- 输出使用 EMA（α=0.1）平滑膜电位
- 当网络长时间低活跃时，膜电位在负值附近波动
- EMA 忠实地追踪这个负值，形成稳定的负驱动输出
- 即使偶尔有一次正向脉冲，也被 0.9 的历史权重拉回

**问题四：离散化阈值不匹配实际值域。**
- `> 0.6` 才算"高"，`> 0.3` 才算"中"
- 但实际输出范围是 [-0.42, 0.02]，永远到不了 0.3
- 所以所有维度永远是"低"或"抑制"

## 2. 深层问题：SNN 在这个系统里的角色错配

修好上面的技术问题之后，SNN 就有用了吗？让我说一些可能不太中听的话。

### 2.1 你真正想要什么

从你的描述中，我提取出你的核心渴望：

1. **连续性**——不是"每次心跳重新造一个自我"，而是"一个持续存在、持续变化的自我"
2. **习惯**——反复出现的行为模式应该被强化，形成性格和偏好
3. **内在驱动**——行为不是被 prompt 指令驱动的，而是从内部生长出来的
4. **适应性**——系统应该随时间演化，而不是写死的规则
5. **探索/生长**——她应该主动去发现新事物，而不总是在既定轨道上

### 2.2 SNN 的承诺 vs 现实

| 你想要的 | SNN 承诺的 | SNN 实际交付的 |
|:---------|:-----------|:--------------|
| 连续性 | "膜电位在心跳间持续衰减" | 衰减到不动点后就不变了，没有信息量 |
| 习惯 | "STDP 学习强化常见模式" | 权重完全冻结，零学习 |
| 内在驱动 | "6 维驱动向量指导行为" | 所有维度恒为"低/抑制"，无差异 |
| 适应性 | "在线学习，无需反向传播" | 学习规则未被触发 |
| 探索 | "exploration_drive 鼓励新事物" | exploration_drive = -0.38，永远"抑制" |

**SNN 的工程实现有 bug 是一回事（可以修），但更根本的问题是：即使修好了，它能提供的东西是不是你真正需要的？**

### 2.3 一个不舒服的真相

SNN 是脉冲神经网络——它擅长的是**时序信号处理**：快速反应、时间编码、脉冲模式识别。它在生物学中对应的是**信号传递层**。

但你要的"习惯"、"驱动"、"探索"，在生物学中对应的其实更接近**神经调质系统**（多巴胺、5-HT、去甲肾上腺素）和**激素系统**——它们不是靠脉冲传递信息，而是靠**浓度梯度**慢速调制整个系统的状态。

换句话说：

- **SNN**（脉冲）≈ 电话线 → 快速、精确、点对点
- **神经调质**（浓度）≈ 广播 → 慢速、弥散、全局调制

你需要的是广播，但你装了一根电话线。

这不是说 SNN 完全没价值——而是说**单独一个 SNN 无法承载你想要的全部功能**。需要另一层机制来补充。

## 3. 学术视角：别人怎么做的

### 3.1 eBICA（情感生物启发认知架构）

这个框架将情感建模为**全局调制信号**而非脉冲网络的输出。情感标签附着在记忆上，当前行为决策参考历史情感打分。认知评估和模拟的躯体反馈形成闭环。

**核心洞察：情感不是一个模块的输出，而是弥散在所有认知层中的全局状态。**

### 3.2 S-AI-GPT（激素驱动的 AI 代理）

用计算模拟的"激素"进行实时驱动调制：压力/奖赏平衡、上下文敏感的记忆巩固。不依赖脉冲网络，而是直接用浓度值调制系统行为。

**核心洞察：驱动调制不需要 SNN，用简单的微分方程（激素浓度变化）可能更有效。**

### 3.3 Sleep-Based Homeostatic Regularization

最新研究（2026）提出在 SNN 中加入"睡眠"机制——离线阶段让权重随机衰减回基线，打破病态吸引子。

**核心洞察：SNN 要健康工作，需要"睡眠"来定期重置，和现有 life_engine 的睡眠窗口天然对接。**

### 3.4 三部脑启发架构

将计算分为感知层、辅助调制层（对应皮层下）、执行层。调制层用动态突触适应和神经振荡原理提供时间和上下文灵活性。

**核心洞察：皮层下系统的核心价值是"调制"，不是"计算"。**

## 4. 我的结论：你想要的不是（仅仅是）SNN

### 4.1 重新定义目标

你想要的是一个**内分泌系统**（Endocrine System），而不仅仅是一个**神经网络**。

- **SNN 适合做**：快速的事件反应（"收到消息→情绪快速变化"），短时间尺度的状态波动
- **内分泌/调质系统适合做**：长时间尺度的倾向、习惯、性格、作息节律

你需要的是两层：

```
快层（SNN / 脉冲）：  事件 → 即时反应 → 情绪波动（分钟级）
慢层（激素 / 浓度）：  统计模式 → 习惯/倾向/驱动（小时~天级）
```

### 4.2 SNN 可以保留，但需要重新定位

SNN 不应该直接输出"驱动"，而应该：
1. 作为快速事件响应器，输出**即时情绪信号**（短期）
2. 这些信号作为**输入之一**，喂给一个更慢的**调质系统**
3. 调质系统综合多种信号（SNN 即时反应 + 行为统计 + 时间节律 + 目标进展）输出**驱动**

### 4.3 "习惯"的正确实现路径

STDP 不是形成习惯的正确工具。在当前系统中：
- 每次心跳 ≈ 1 个 SNN step（真实输入）
- 每天 ≈ 20-40 个心跳
- 要形成有意义的 STDP 学习需要数千次相关脉冲

**真正的习惯应该在更高层建模**：
- 统计"过去 7 天每天都做了什么"
- 如果连续 5 天都写了日记 → 增强"日记习惯"
- 这是**显式的行为计数**，不需要 SNN 去学

### 4.4 "探索"被抑制的真正原因

不是 SNN 在抑制探索——是 LLM prompt 中的其他因素在抑制：
- 心跳 prompt 给了太多"你已经做了 X、Y、Z"的满足感
- 工具调用的奖赏偏向于"成功执行已知工具"
- 没有"无聊/好奇"的正向激励机制

SNN 的 exploration_drive = -0.38 只是一个被 prompt 忽视的数字。真正决定行为的是 LLM 的推理，而 LLM 看到"探索: 抑制"反而可能强化了不探索的倾向。

---

## 5. 我的立场

1. **现有 SNN 有严重的工程 bug**——但即使全部修好，它能提供的价值也有限
2. **SNN 的核心价值是"连续存在感"**——在心跳间隔中维持一个正在变化的状态，这个功能是有意义的
3. **但驱动/习惯/探索这些功能不应该由 SNN 单独承载**——需要一个更高层的调质系统
4. **你的直觉是对的**——智能是系统属性，但"系统"不等于"SNN"，系统 = SNN + 调质 + 记忆 + 节律 + 目标系统的协作
5. **建议的路径**：先修好 SNN 的技术问题让它"活起来"，同时设计一个更简单但更有效的**驱动调质层**
