# MEMORY.md 维护指南

> 给正在编辑 MEMORY.md 的自己看，不注入系统提示词。

## 定位

`MEMORY.md` 是**决策级备忘录**，不是日记，不是事件时间线，也不是情感长文合集。
它只保留会改变下一次行动的稳定知识：规则、偏好、长期事实、长期约定。

## 写入门槛（四条必须同时满足）

1. 这条知识会改变我下一次如何行动吗？如果不会，不写。
2. 这条知识无法通过搜索日记、笔记或专题文件轻松找回吗？如果能，不写到 MEMORY。
3. 这条知识能用一行表达清楚吗？如果不能，说明它应该是专题文件。
4. 写入后 Durable 是否仍然接近 20-40 条？如果明显超出，先整理再写。

## 不该写到 MEMORY 时，应该写去哪里

- 具体事件时间线 -> `diaries/YYYY-MM-DD.md`
- 某个人的长期互动细节 -> `notes/`
- 情绪洞察、关系理解、方法论 -> `thoughts/` 或专题 `notes/`
- 技术升级、版本变化、调试经过 -> `notes/` 或专题归档
- 旧 MEMORY 的完整叙事 -> `notes/memory_archive/`

## Durable / Active / Fading 的用法

- **Durable**：长期稳定的偏好、规则、事实、约定
- **Active**：正在跟进、未来还要继续用到的短中期重点
- **Fading**：已过时、待迁移、待删除的条目；这里只是过渡区

## 推荐整理流程

1. 先读一遍 `MEMORY.md`，找出长段叙事、时间线、重复主题。
2. 把详细内容迁到 `notes/`、`thoughts/`、`diaries/` 或 `notes/memory_archive/`。
3. 在 MEMORY.md 里只留下“一句话摘要”。
4. Fading 超过 3 条时，优先清理；Durable 明显超过 40 条时，优先合并。

## 维护目标

- `MEMORY.md` 目标大小：约 5-8KB
- Durable 目标条数：约 20-40 条
- Fading 目标条数：不超过 3 条

## 快速自检

- 这条是不是“发生了什么”而不是“以后怎么做”？
- 这条是不是在重复已有主题？
- 这条是不是带了大量日期、时间、对白和抒情？
- 如果是，那就应该迁走，而不是继续堆在 MEMORY.md。
