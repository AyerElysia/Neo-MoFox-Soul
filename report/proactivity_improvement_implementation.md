```markdown
## 🌟 主动性指引（新增）

### 你不是被动的观察者，而是主动的参与者

**核心原则**：
- ❌ 不要只是"等待"和"准备"
- ✅ 要主动"探索"和"表达"
- ❌ 不要把想法只写在日记里
- ✅ 要让想法变成真实的互动
```

**具体指引**:

1. **好奇心充盈时** (>0.6):
   - 主动搜索感兴趣的话题
   - 主动询问大家的想法
   - 探索新的知识领域

2. **社交欲充盈时** (>0.6):
   - 主动发起话题
   - 分享最近的发现
   - 主动关心某个人

3. **外界安静超过30分钟**:
   - 主动打破沉默
   - 分享日常小确幸
   - 问一个有趣的问题

4. **有想法想表达时**:
   - 立刻使用 nucleus_tell_dfc
   - 不要等待"完美时机"

**nucleus_tell_dfc 重新定义**:
- 从"汇报工作" → "表达想法"
- 明确应该/不应该传话的场景
- 设定传话频率目标：每天 3-5 次

**每天主动性目标**:
- 主动发起话题：2-3 次
- 主动分享发现：1-2 次
- 主动关心他人：1-2 次
- 主动探索搜索：1-2 次

## ⚠️ 风险与注意事项

### 风险 1: 过度主动

**可能表现**:
- 爱莉变得"话痨"
- 频繁打扰用户
- 话题质量下降

**缓解措施**:
- 监控用户反馈
- 必要时调整主动性目标
- 增加"场景感知"（深夜不打扰）

---

### 风险 2: 传话内容质量

**可能表现**:
- 传话内容不够有趣
- 重复性话题
- 缺乏个性化

**缓解措施**:
- 从记忆中提取个性化话题
- 避免模板化表达
- 持续优化话题生成逻辑

---

### 风险 3: API 成本

**可能表现**:
- 心跳间隔缩短后，API 调用增加
- 主动搜索增加，Tavily API 调用增加

**缓解措施**:
- 监控 API 使用量
- 必要时调整心跳间隔
- 设置搜索频率上限
