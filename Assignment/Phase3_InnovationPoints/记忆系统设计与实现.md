# 记忆系统设计与实现：仿生记忆的完整架构

> **核心定位**：Neo-MoFox实现了完整的仿生记忆系统——激活扩散（Spreading Activation）、Hebbian学习（共同激活强化）、软遗忘（Ebbinghaus曲线），形成真正的记忆而非简单的向量检索库。

## 一、仿生记忆系统的设计哲学

### 1.1 传统AI的"记忆"局限

**传统向量检索库的问题**：

```
传统向量库设计：
┌─────────────────────────────┐
│  文本 → Embedding → 向量库    │
│  查询 → Embedding → 相似度搜索 │
│  返回 Top-K 文档              │
│                              │
│  问题：                       │
│  - 无联想机制（只有相似度）     │
│  - 无记忆衰减（永久存储）       │
│  - 无连接建立（孤立节点）       │
│  - 无情感保护（情感记忆无特权） │
└─────────────────────────────┘
```

**与人类记忆的对比**：

| 维度 | 传统向量库 | 人类记忆 |
|-----|-----------|---------|
| **联想机制** | 只有相似度 | 激活扩散（从A想起B） |
| **记忆衰减** | 无（永久） | 遗忘曲线（不回忆则遗忘） |
| **连接建立** | 无（孤立） | 共激活强化连接 |
| **情感影响** | 无 | 情感记忆更强 |
| **重要性影响** | 无 | 重要记忆更持久 |
| **检索练习效应** | 无 | 回忆越多越牢固 |

### 1.2 Neo-MoFox的仿生记忆设计

**认知科学理论基础**：

1. **激活扩散理论（Spreading Activation）**：
   - Collins & Loftus (1975)
   - 从激活节点出发，激活沿连接扩散
   - 例如：从"医生"想起"护士"、"医院"

2. **Hebbian学习理论**：
   - Donald Hebb (1949)
   - "Together fired, wired together"
   - 共激活的神经元之间建立强连接

3. **遗忘曲线理论**：
   - Hermann Ebbinghaus (1885)
   - 记忆强度随时间衰减
   - 回忆（提取练习）强化记忆

4. **情感增强记忆**：
   - 情感唤醒度高的记忆更持久
   - Cahill & McGaugh (1998)

**Neo-MoFox的记忆系统架构**：

```
┌──────────────────────────────────────┐
│       记忆节点（Memory Nodes）         │
│       - 文件节点、事件节点              │
│       - 激活强度、情感唤醒度            │
│       - 访问计数、重要性                │
└────────────┬─────────────────────────┘
             │
             │ 激活扩散
             │

│       记忆连接（Memory Edges）         │

             │
             │ Hebbian 学习
             │（共同激活 → 强化连接）
┌────────────┴─────────────────────────┐
│       检索与整合                       │
│       - 向量检索（ChromaDB）           │
│       - 全文搜索（FTS5）               │
│       - RRF融合                        │
│       - 激活扩散联想                   │
└────────────┬─────────────────────────┘
             │
             │ 记忆衰减
             │
┌────────────┴─────────────────────────┐
```

## 三、混合检索机制详解

### 3.1 四层检索架构

**检索流程**：

```
用户查询 → 向量检索 + FTS → RRF融合 → 激活扩散 → Hebbian学习
```

**完整检索代码**：

```python
# plugins/life_engine/memory/search.py::search_memory

async def search_memory(
    db: sqlite3.Connection,
    query: str,
    collection: Any,
    top_k: int = 5,
    enable_association: bool = True,
    file_types: list[str] | None = None,
    time_range_days: int = 0,
    emit_visual_event: Any = None,
    increment_access_func: Any = None,
    reinforce_coactivated_func: Any = None,
) -> list[SearchResult]:
    """混合检索 + 联想扩散。"""
    results: list[SearchResult] = []

    # 1. 向量检索（语义相似度）
    vector_results = await vector_search(query, collection, top_k=10)

    # 2. 全文搜索（关键词匹配）
    fts_results = await fts_search(db, query, top_k=10)

    # 3. RRF融合（Reciprocal Rank Fusion）
    fused_results = await rrf_fusion(vector_results, fts_results, k=RRF_K)

    # 4. 激活扩散联想（从Top-K节点出发扩散）
    if enable_association and fused_results:
        seed_ids = [node_id for node_id, _ in fused_results[:3]]
        association_results = await spread_activation(
            db=db,
            seed_ids=seed_ids,
            max_depth=2,
            max_results=5,
            spread_decay=SPREAD_DECAY,
            spread_threshold=SPREAD_THRESHOLD,
        )
        fused_results.extend(association_results)

    # 5. 过滤与排序
    final_results = await filter_results(
        db=db,
        fused_results=fused_results,
        file_types=file_types,
        time_range_days=time_range_days,
    )

    # 6. 构建SearchResult对象
    for node_id, relevance, path, reason in final_results[:top_k]:
        node = await get_node_by_id(db, node_id)
        if not node:
            continue

        snippet = await get_snippet(db, node_id)

        result = SearchResult(
            file_path=node.file_path,
            title=node.title,
            snippet=snippet,
            relevance=relevance,
            source="direct" if relevance > 0.5 else "associated",
            association_path=path if path else [],
            association_reason=reason,
        )
        results.append(result)

        # 7. 记录访问（提取练习效应）
        if increment_access_func:
            await increment_access_func(node_id)

    # 8. Hebbian学习（共激活强化）
    if reinforce_coactivated_func and results:
        activated_ids = [r.file_path for r in results[:3]]
        await reinforce_coactivated_func(activated_ids)

    return results
```

### 3.2 向量检索（Vector Search）

**实现细节**：

```python
async def vector_search(
    query: str,
    collection: Any,
    top_k: int = 10,
) -> list[tuple[str, float]]:
    """向量相似度检索。"""
    # 生成查询embedding
    query_embedding = await embed_text(query)

    # ChromaDB查询
    results = collection.query(
        query_embeddings=[query_embedding],
        n_results=top_k,
        include=["distances"],
    )

    # 转换距离为相似度
    raw_pairs: list[tuple[str, float]] = []
    for node_id, distance in zip(results["ids"][0], results["distances"][0]):
        # ChromaDB返回距离（越小越相似）
        # 转换为相似度：similarity = 1 / (1 + distance)
        similarity = 1.0 / (1.0 + distance)
        raw_pairs.append((node_id, similarity))

    # 过滤脏节点ID（向量库存在但节点表不存在）
    filtered_pairs = await filter_existing_nodes(raw_pairs)

    return filtered_pairs
```

**向量库同步机制**：

```python
async def sync_embedding(
    db: sqlite3.Connection,
    collection: Any,
    file_path: str,
    content: str,
):
    """同步文件内容到向量库。"""
    # 获取节点
    node = await get_node_by_file_path(db, file_path)
    if not node:
        return

    # 生成embedding（截断到3000字符）
    embedding = await embed_text(content[:3000])

    # Upsert到ChromaDB
    collection.upsert(
        ids=[node.node_id],
        embeddings=[embedding],
        documents=[content[:500]],  # 存储摘要（500字符）
        metadatas=[
            {
                "file_path": file_path,
                "title": node.title,
                "created_at": node.created_at,
            }
        ],
    )

    # 更新节点表标记
    cursor = db.cursor()
    cursor.execute(
        "UPDATE memory_nodes SET embedding_synced = 1 WHERE node_id = ?",
        (node.node_id,)
    )
    db.commit()
```

**FTS搜索实现**：

```python
async def fts_search(
    db: sqlite3.Connection,
    query: str,
    top_k: int = 10,
) -> list[tuple[str, float]]:
    """全文搜索。"""
    cursor = db.cursor()

    # FTS5查询（MATCH操作符）
    cursor.execute(
        """
        SELECT node_id, title, content, rank
        FROM memory_fts
        WHERE memory_fts MATCH ?
        ORDER BY rank
        LIMIT ?
        """,
        (query, top_k),
    )

    results: list[tuple[str, float]] = []
    for row in cursor.fetchall():
        node_id = row[0]
        rank = row[3]

        # FTS5 rank转换为相似度（BM25-like）
        # rank越小 → 相似度越高
        similarity = 1.0 / (1.0 + abs(rank) / 10.0)

        results.append((node_id, similarity))

    return results
```

### 3.4 RRF融合（Reciprocal Rank Fusion）

**RRF算法原理**：

```
RRF_score = Σ (1 / (k + rank_i))

其中：
- k: 融合参数（默认60）
- rank_i: 在第i个检索系统中的排名

例如：
节点A：向量检索 rank=1, FTS rank=3
  RRF_score = 1/(60+1) + 1/(60+3) ≈ 0.016 + 0.015 = 0.031

节点B：向量检索 rank=5, FTS rank=1
  RRF_score = 1/(60+5) + 1/(60+1) ≈ 0.015 + 0.016 = 0.031

节点C：向量检索 rank=10, FTS rank=无
  RRF_score = 1/(60+10) = 0.014
```

**实现代码**：

```python
async def rrf_fusion(
    vector_results: list[tuple[str, float]],
    fts_results: list[tuple[str, float]],
    k: int = 60,
) -> list[tuple[str, float]]:
    """RRF融合向量检索和全文搜索结果。"""
    # 构建排名字典
    ranks: dict[str, list[int]] = defaultdict(list)

    for rank, (node_id, _) in enumerate(vector_results, start=1):
        ranks[node_id].append(rank)

    for rank, (node_id, _) in enumerate(fts_results, start=1):
        ranks[node_id].append(rank)

    # 计算RRF分数
    fused: list[tuple[str, float]] = []
    for node_id, rank_list in ranks.items():
        rrf_score = sum(1.0 / (k + rank) for rank in rank_list)
        fused.append((node_id, rrf_score))

    # 按RRF分数降序排序
    fused.sort(key=lambda x: x[1], reverse=True)

    return fused
```

**RRF的优势**：

| 维度 | 传统融合方法 | RRF融合 |
|-----|-------------|---------|
| **参数敏感性** | 需调权重（向量权重vs FTS权重） | 无需调权重（自动平衡） |
| **缺失值处理** | 需手动处理 | 自动处理（单系统也能融合） |
| **排序稳定性** | 对权重敏感 | 对k参数不敏感 |
| **计算复杂度** | 低 | 低（O(n log n)) |

### 3.5 激活扩散联想

**算法原理**：

```
从种子节点出发，沿连接扩散激活：

激活值 = 种子激活 × decay^depth

例如：
种子A（激活=1.0）
  → 连接到B（权重=0.6）
    → B激活 = 1.0 × 0.6 × 0.7^1 = 0.42
    → 连接到C（权重=0.5）
      → C激活 = 0.42 × 0.5 × 0.7^2 = 0.10

扩散深度控制联想范围：
- depth=1: 直接联想（一步）
- depth=2: 二级联想（两步）
- depth=3: 三级联想（三步，远距离联想）
```

**实现代码**：

```python
async def spread_activation(
    db: sqlite3.Connection,
    seed_ids: list[str],
    max_depth: int = 2,
    max_results: int = 10,
    spread_decay: float = 0.7,
    spread_threshold: float = 0.3,
) -> list[tuple[str, float, list[str], str]]:
    """激活扩散联想。"""
    # 激活值字典
    activations: dict[str, float] = {}
    # 路径字典（记录激活路径）
    paths: dict[str, list[str]] = defaultdict(list)

    # 初始化种子节点（激活值=1.0）
    for seed_id in seed_ids:
        activations[seed_id] = 1.0
        paths[seed_id] = [seed_id]

    # BFS扩散
    current_depth = 0
    current_activations = dict(activations)

    while current_depth < max_depth:
        next_activations: dict[str, float] = {}

        for node_id, activation in current_activations.items():
            # 获取从该节点出发的边
            edges = await get_edges_from(db, node_id, min_weight=spread_threshold)

            for edge in edges:
                # 计算扩散激活值
                spread_activation = activation * edge.weight * (spread_decay ** current_depth)

                # 如果目标节点激活值更高，则更新
                target_id = edge.target_id
                if target_id not in activations or spread_activation > activations.get(target_id, 0):
                    next_activations[target_id] = spread_activation
                    activations[target_id] = spread_activation

                    # 记录激活路径
                    paths[target_id] = paths[node_id] + [target_id]

        # 移动到下一层
        current_activations = next_activations
        current_depth += 1

        # 如果没有新的激活节点，停止扩散
        if not current_activations:
            break

    # 排序并返回Top-K
    sorted_activations = sorted(activations.items(), key=lambda x: x[1], reverse=True)

    # 过滤掉种子节点（只返回联想节点）
    results: list[tuple[str, float, list[str], str]] = []
    for node_id, activation in sorted_activations[:max_results + len(seed_ids)]:
        if node_id not in seed_ids:
            # 获取激活路径
            path = paths[node_id]
            reason = f"联想激活（路径长度={len(path)-1}）"
            results.append((node_id, activation, path, reason))

    return results[:max_results]
```

## 四、Hebbian学习机制详解

### 4.1 共激活强化原理

**Hebbian学习规则**：

```
当节点A和节点B在同一检索中被激活 → 强化A-B连接

Δweight = learning_rate × coactivation_strength

例如：
节点A（激活=0.8）和节点B（激活=0.6）共激活
  coactivation_strength = 0.8 × 0.6 = 0.48
  Δweight = 0.1 × 0.48 = 0.048

如果A-B边初始weight=0.5 → 强化后weight=0.548
```

**实现代码**：

```python
# plugins/life_engine/memory/edges.py::reinforce_coactivated

async def reinforce_coactivated(
    db: sqlite3.Connection,
    node_ids: list[str],
    learning_rate: float = 0.1,
):
    """Hebbian学习：强化共激活节点之间的连接。"""
    if len(node_ids) < 2:
        return

    # 获取节点激活强度
    cursor = db.cursor()
    activations: dict[str, float] = {}
    for node_id in node_ids:
        cursor.execute(
            "SELECT activation_strength FROM memory_nodes WHERE node_id = ?",
            (node_id,),
        )
        row = cursor.fetchone()
        if row:
            activations[node_id] = float(row[0])

    # 遍历所有共激活节点对
    for i, source_id in enumerate(node_ids):
        for target_id in node_ids[i+1:]:
            # 计算共激活强度
            source_activation = activations.get(source_id, 0.5)
            target_activation = activations.get(target_id, 0.5)
            coactivation_strength = source_activation * target_activation

            # 强化连接
            await create_or_update_edge(
                db=db,
                source_id=source_id,
                target_id=target_id,
                edge_type=EdgeType.ASSOCIATES,
                reason="Hebbian共激活强化",
                strength_change=learning_rate * coactivation_strength,
                bidirectional=True,
            )
```

**连接更新逻辑**：

```python
async def create_or_update_edge(
    db: sqlite3.Connection,
    source_id: str,
    target_id: str,
    edge_type: EdgeType,
    reason: str = "",
    strength_change: float = 0.0,
    bidirectional: bool = True,
) -> MemoryEdge:
    """创建或更新边（应用Hebbian学习）。"""
    cursor = db.cursor()

    edge_id = f"edge_{source_id}_{target_id}_{edge_type.value}"

    # 检查边是否存在
    cursor.execute(
        "SELECT * FROM memory_edges WHERE edge_id = ?",
        (edge_id,),
    )
    row = cursor.fetchone()

    if row:
        # 边已存在 → 强化
        edge = row_to_edge(row)

        # 更新reinforcement（Hebbian强化累积）
        new_reinforcement = edge.reinforcement + strength_change

        # 更新权重（base_strength + reinforcement）
        new_weight = edge.base_strength + new_reinforcement

        # 更新共激活计数
        new_activation_count = edge.activation_count + 1

        # 更新最后激活时间
        last_activated_at = time.time()

        cursor.execute(
            """
            UPDATE memory_edges
            SET reinforcement = ?, weight = ?, activation_count = ?, last_activated_at = ?
            WHERE edge_id = ?
            """,
            (new_reinforcement, new_weight, new_activation_count, last_activated_at, edge_id),
        )
    else:
        # 边不存在 → 创建
        now = time.time()
        cursor.execute(
            """
            INSERT INTO memory_edges (
                edge_id, source_id, target_id, edge_type, weight, base_strength,
                reinforcement, activation_count, last_activated_at, reason, created_at, bidirectional
            ) VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
            """,
            (
                edge_id, source_id, target_id, edge_type.value,
                0.5 + strength_change, 0.5, strength_change, 1, now, reason, now, 1 if bidirectional else 0
            ),
        )

    db.commit()
```

## 五、记忆衰减机制详解

### 5.1 Ebbinghaus遗忘曲线

**遗忘曲线公式**：

```
记忆强度 = exp(-λ × days_since_last_access)

其中：
- λ: 遗忘衰减系数（Neo-MoFox使用λ=0.05）
- days_since_last_access: 最后访问距今的天数

遗忘曲线特性：
- λ=0.05 → 半衰期 ≈ 14天
- 14天后，记忆强度衰减到 50%
- 30天后，记忆强度衰减到 22%
```

**Ebbinghaus曲线可视化**：

```
记忆强度 vs 时间（天）
1.00 ──────╮
           │╲
0.75       │ ╲
           │  ╲
0.50       │   ╲──────╮ (14天，半衰期)
           │    ╲     │
0.25       │     ╲    │╲
           │      ╲   │ ╲
0.00 ──────┴───────╲──┴──╲───
           0   7  14  21  30  天

λ=0.05时的遗忘曲线
```

### 5.2 多种保护因素

**记忆强度计算公式**：

```python
# plugins/life_engine/memory/decay.py::compute_memory_strength

def compute_memory_strength(
    node: MemoryNode,
    decay_lambda: float = 0.05,
) -> float:
    """计算记忆强度，结合多种保护因素。"""
    if not node.last_accessed_at:
        return node.activation_strength

    now = time.time()
    days_since = (now - node.last_accessed_at) / 86400

    # 1. 基础时间衰减（Ebbinghaus）
    time_decay = math.exp(-decay_lambda * days_since)

    # 2. 提取练习效应（Testing Effect）
    # 回忆次数越多 → 记忆越牢固
    retrieval_bonus = math.log(1 + node.access_count) * 0.1

    # 3. 情感增强记忆（Emotional Enhancement）
    # 情感唤醒度高的记忆更持久
    emotional_shield = node.emotional_arousal * 0.2

    # 4. 重要性保护
    # 重要性高的记忆衰减更慢
    importance_shield = node.importance * 0.1

    # 最终强度 = 基础衰减 + 各种保护因素
    strength = time_decay + retrieval_bonus + emotional_shield + importance_shield

    return min(max(strength, 0.0), 1.0)
```

### 5.3 记忆衰减的应用

**定期衰减任务**：

```python
# plugins/life_engine/memory/decay.py::apply_decay

async def apply_decay(db: sqlite3.Connection) -> int:
    """应用遗忘衰减（定期任务，每日一次）。"""
    cursor = db.cursor()

    # 获取所有节点
    cursor.execute("SELECT * FROM memory_nodes")
    rows = cursor.fetchall()

    updated = 0
    for row in rows:
        node = row_to_node(row)

        # 计算新的记忆强度
        new_strength = compute_memory_strength(node)

        # 如果强度变化显著，更新节点
        if abs(new_strength - node.activation_strength) > 0.01:
            cursor.execute(
                "UPDATE memory_nodes SET activation_strength = ? WHERE node_id = ?",
                (new_strength, node.node_id),
            )
            updated += 1

    # 边衰减（ASSOCIATES边）
    cursor.execute(
        "SELECT * FROM memory_edges WHERE edge_type = ?",
        (EdgeType.ASSOCIATES.value,),
    )
    for row in cursor.fetchall():
        edge = row_to_edge(row)

        if edge.last_activated_at:
            days_since = (time.time() - edge.last_activated_at) / 86400
            decay_factor = math.exp(-DECAY_LAMBDA * days_since)

            # 边权重衰减
            new_weight = edge.base_strength + edge.reinforcement * decay_factor

            # 如果权重低于阈值，删除边（剪枝）
            if new_weight < PRUNE_THRESHOLD:
                cursor.execute("DELETE FROM memory_edges WHERE edge_id = ?", (edge.edge_id,))
            elif abs(new_weight - edge.weight) > 0.01:
                cursor.execute(
                    "UPDATE memory_edges SET weight = ? WHERE edge_id = ?",
                    (new_weight, edge.edge_id),
                )

    db.commit()
    logger.info(f"遗忘衰减完成，更新了 {updated} 个节点")
    return updated
```

**边剪枝机制**：

```python
# 弱连接删除
PRUNE_THRESHOLD = 0.1

if new_weight < 0.1:
    # 连接过弱 → 删除（清理无效连接）
    cursor.execute("DELETE FROM memory_edges WHERE edge_id = ?", (edge.edge_id,))
```

**剪枝的意义**：

- **清理无效连接**：避免记忆网络过于稠密（影响检索效率）
- **保留强连接**：只保留真正有意义的联想路径
- **动态平衡**：新连接建立 + 弱连接剪枝 → 网络结构优化

```python
# plugins/life_engine/snn/core.py::DriveCoreNetwork

def replay_episodes(
    self,
    episodes: list[list[np.ndarray]],
    speed_multiplier: float = 5.0,
    reward_signal: float = 0.0,
) -> dict[str, Any]:
    """NREM阶段：回放历史事件片段（通过SNN）。"""
    stats = {
        "steps": 0,
        "total_episodes": len(episodes),
    }

    for episode in episodes:
        # 按时间压缩回放（speed_multiplier=5 → 5倍速）
        for features in episode:
            # SNN真实输入步（带学习）
            self.step(features, reward=reward_signal)
            stats["steps"] += 1

    return stats
```

- **事件权重调整**：重要事件多次回放 → 强化权重
