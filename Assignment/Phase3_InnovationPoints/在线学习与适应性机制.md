# 在线学习与适应性：SNN实时演化的驱动系统

> **核心定位**：Neo-MoFox的SNN系统实现了真正的在线学习——每次心跳都会根据事件流调整神经网络参数，形成持续演化的内在驱动系统，而非传统AI的静态参数冻结。

---

## 一、在线学习的设计哲学

### 1.1 传统AI的学习局限

**传统LLM的学习模式**：

```
训练阶段（Pre-training）：
┌─────────────────────────────────┐
│  大规模数据训练（数月）          │
│  参数冻结 → 固定能力              │
│  无法在运行中学习                 │
└─────────────────────────────────┘

部署阶段（Deployment）：
┌─────────────────────────────────┐
│  参数完全冻结                     │
│  每次对话独立                     │
│  无法从对话历史中学习             │
│  无法持续成长                     │
└─────────────────────────────────┘
```

**问题**：

- **无法在线学习**：部署后无法从实际对话中学习新模式
- **无法持续成长**：能力冻结在训练完成时
- **无法个性化适应**：对不同用户、不同场景无法动态调整
- **缺乏经验积累**：每次对话都是"第一次"

### 1.2 Neo-MoFox的在线学习设计

**SNN的持续学习模式**：

```
初始化阶段：
┌─────────────────────────────────┐
│  SNN 参数初始化                   │
│  突触权重随机 → 稳态基线          │
│  软 STDP 学习规则                 │
└─────────────────────────────────┘

运行阶段（24/7）：
┌─────────────────────────────────┐
│  每次心跳 → 真实输入步             │
│  事件流 → 特征提取 → SNN输入       │
│  Reward信号 → STDP权重调整         │
│  稳态调节 → 参数归一化             │
│  持续演化 → 能力成长               │
└─────────────────────────────────┘
```

**关键特性**：

- **在线学习**：每次心跳都会学习（真实输入步）
- **持续演化**：突触权重随时间持续调整
- **个性化适应**：根据实际事件流调整驱动产生模式
- **经验积累**：SNN参数记录了长期交互经验

## 二、SNN在线学习机制详解

### 2.1 真实输入步（Real Step）

**概念区分**：

| 步类型 | 定义 | 学习 | 频率 |
|------|-----|------|------|
| **衰减步（Decay Only）** | 只做膜电位衰减、trace衰减 | 不学习 | 每10秒（独立tick循环） |
| **真实输入步（Real Step）** | 接收外部输入、计算驱动、调整权重 | 学习 | 每30秒（心跳前） |

**实现代码**：

```python
# plugins/life_engine/snn/core.py::DriveCoreNetwork

def step(self, features: np.ndarray, reward: float = 0.0) -> np.ndarray:
    """真实输入步（带学习的完整更新）。

    Args:
        features: 8维特征向量（从事件流提取）
        reward: 奖赏信号（从最近事件结果计算）

    Returns:
        6维驱动向量
    """
    # 输入层脉冲（根据特征强度产生脉冲）
    input_spikes = self._input_layer.generate_spikes(features)

    # 隐藏层激活（LIF神经元动力学）
    hidden_spikes, hidden_voltages = self._hidden_layer.step(input_spikes)

    # 输出层驱动（根据隐藏层脉冲计算驱动）
    output_rates = self._output_layer.compute_rates(hidden_spikes)

    # STDP学习（突触权重调整）
    if reward != 0.0 or self._real_step_count > 0:
        self.syn_in_hid.apply_stdp(
            pre_traces=self._input_layer.traces,
            post_traces=self._hidden_layer.traces,
            reward=reward,
        )
        self.syn_hid_out.apply_stdp(
            pre_traces=self._hidden_layer.traces,
            post_rates=output_rates,
            reward=reward,
        )

    self._real_step_count += 1
    self.tick_count += 1

    return output_rates
```

**真实输入步的完整流程**：

```
事件流 → 特征提取 → SNN输入 → 脉冲传播 → 驱动输出 → STDP学习
```

### 2.2 事件流特征提取

**特征向量的8个维度**：

```python
# plugins/life_engine/snn/bridge.py::SNNBridge

def extract_features_from_events(
    self,
    events: list[LifeEngineEvent],
    window_seconds: float = 1800.0,
) -> np.ndarray:
    """从事件流提取8维特征向量。

    特征维度：
    1. msg_in_rate        - 入站消息频率
    2. msg_out_rate       - 出站消息频率
    3. tool_call_rate     - 工具调用频率
    4. tool_success_rate  - 工具成功率
    5. heartbeat_rate     - 心跳独白频率
    6. idle_rate          - 空转心跳比例
    7. silence_duration   - 沉默时长（分钟）
    8. emotional_intensity - 情绪强度（调质层方差）

    Returns:
        np.ndarray: 8维特征向量，每维 ∈ [0, 1]
    """
```

**特征计算逻辑**：

```python
features = np.zeros(8, dtype=np.float64)

now = time.time()
window_start = now - window_seconds
window_events = [e for e in events if e.timestamp >= window_start]

# 1. msg_in_rate（入站消息频率）
msg_in_count = len([
    e for e in window_events
    if e.event_type == EventType.MESSAGE
    and "入站" in str(e.source_detail or "")
])
msg_in_rate = min(msg_in_count / 10.0, 1.0)  # 10条/半小时 = 1.0
features[0] = msg_in_rate

# 2. msg_out_rate（出站消息频率）
msg_out_count = len([
    e for e in window_events
    if e.event_type == EventType.MESSAGE
    and "出站" in str(e.source_detail or "")
])
msg_out_rate = min(msg_out_count / 10.0, 1.0)
features[1] = msg_out_rate

# 3. tool_call_rate（工具调用频率）
tool_call_count = len([
    e for e in window_events
    if e.event_type == EventType.TOOL_CALL
])
tool_call_rate = min(tool_call_count / 15.0, 1.0)  # 15次/半小时 = 1.0
features[2] = tool_call_rate

# 4. tool_success_rate（工具成功率）
tool_results = [
    e for e in window_events
    if e.event_type == EventType.TOOL_RESULT
]
if tool_results:
    success_count = len([e for e in tool_results if e.tool_success])
    success_rate = success_count / len(tool_results)
    features[3] = success_rate

# 5. heartbeat_rate（心跳独白频率）
heartbeat_count = len([
    e for e in window_events
    if e.event_type == EventType.HEARTBEAT
])
heartbeat_rate = min(heartbeat_count / 60.0, 1.0)  # 60次/半小时 = 1.0
features[4] = heartbeat_rate

# 6. idle_rate（空转心跳比例）
idle_count = len([
    e for e in window_events
    if e.event_type == EventType.HEARTBEAT
    and getattr(e, "is_idle", False)
])
if heartbeat_count > 0:
    idle_rate = idle_count / heartbeat_count
    features[5] = idle_rate

# 7. silence_duration（沉默时长）
last_external_msg_time = None
for e in reversed(events):
    if e.event_type == EventType.MESSAGE:
        source = str(e.source or "")
        if source != "life_engine":
            last_external_msg_time = e.timestamp
            break

if last_external_msg_time is not None:
    silence_seconds = now - last_external_msg_time
    silence_minutes = silence_seconds / 60.0
    silence_feature = min(silence_minutes / 60.0, 1.0)  # 60分钟 = 1.0
    features[6] = silence_feature

# 8. emotional_intensity（情绪强度）
if self._service._inner_state is not None:
    modulators = self._service._inner_state.modulators
    values = [mod.value for mod in modulators.get_all()]
    if values:
        variance = np.var(values)
        intensity = min(variance * 4.0, 1.0)  # variance=0.25 → intensity=1.0
        features[7] = intensity

return features
```

### 2.3 Reward信号计算

**Reward的作用**：指导STDP学习的方向（强化或抑制特定连接）。

**计算逻辑**：

```python
# plugins/life_engine/snn/bridge.py::SNNBridge

def record_heartbeat_result(
    self,
    tool_event_count: int,
    tool_success_count: int,
    tool_fail_count: int,
    idle_count: int,
    had_text_reply: bool,
) -> float:
    """记录心跳结果，计算reward信号。

    Reward计算规则：
    - 工具成功 → 正reward（强化成功策略）
    - 工具失败 → 负reward（抑制失败策略）
    - 空转过多 → 负reward（抑制空转）
    - 有文本回复 → 正reward（强化社交）

    Returns:
        float: reward信号 ∈ [-1, 1]
    """
    reward = 0.0

    # 工具成功率影响
    if tool_event_count > 0:
        success_rate = tool_success_count / tool_event_count
        reward += 0.3 * (success_rate - 0.5)  # 成功率高于50% → 正reward

    # 工具失败惩罚
    if tool_fail_count > 0:
        fail_penalty = min(tool_fail_count / 5.0, 0.2)
        reward -= fail_penalty

    # 空转惩罚（idle超过阈值）
    if idle_count > 5:
        idle_penalty = min((idle_count - 5) / 10.0, 0.15)
        reward -= idle_penalty

    # 社交奖励（产生文本回复）
    if had_text_reply:
        reward += 0.2

    # 裁剪到 [-1, 1]
    reward = max(-1.0, min(1.0, reward))

    # 记录到历史
    self._reward_history.append(reward)
    self._reward_history = self._reward_history[-100:]  # 保留最近100次

    return reward
```

**Reward示例**：

| 场景 | tool成功 | tool失败 | idle | 回复 | Reward |
|------|---------|---------|------|------|--------|
| **活跃对话** | 8次成功 | 1次失败 | 2次 | 有 | +0.4 |
| **工具探索** | 5次成功 | 3次失败 | 0次 | 无 | +0.1 |
| **长时间沉默** | 0次 | 0次 | 10次 | 无 | -0.15 |
| **梦中整理** | 2次成功 | 0次 | 0次 | 无 | +0.1 |

## 三、STDP学习机制详解

### 3.1 软STDP规则

**传统STDP（硬STDP）**：

```python
# 生物学STDP规则（不可直接用于在线学习）
if pre_spike_time < post_spike_time:
    # LTP（长时程增强）
    Δw = A_plus * exp(-(post_time - pre_time) / tau_plus)
else:
    # LTD（长时程抑制）
    Δw = -A_minus * exp(-(pre_time - post_time) / tau_minus)

# 问题：
# 1. 权重无约束 → 可能无限增长或衰减
# 2. 无稳态调节 → 网络失去平衡
# 3. 对reward不敏感 → 无法引导学习方向
```

**Neo-MoFox的软STDP**：

```python
# plugins/life_engine/snn/core.py::STDPSynapse

def apply_stdp(
    self,
    pre_traces: np.ndarray,
    post_traces: np.ndarray,
    reward: float = 0.0,
):
    """应用软STDP学习规则。

    核心机制：
    1. 基于trace的权重调整（而非精确时间）
    2. Reward调制（奖励信号影响调整方向）
    3. 权重约束（限制在 [w_min, w_max]）
    4. 稳态归一化（防止权重无限增长）

    Args:
        pre_traces: 前脉冲trace（记录近期脉冲历史）
        post_traces: 后脉冲trace
        reward: 奖赏信号 ∈ [-1, 1]
    """
    # LTP成分（前trace × 后trace）
    ltp = np.outer(pre_traces, post_traces)

    # LTD成分（前trace × 后静默）
    ltd = np.outer(pre_traces, 1.0 - post_traces)

    # Reward调制
    # reward > 0 → 强化LTP（奖励强化路径）
    # reward < 0 → 强化LTD（惩罚抑制路径）
    if reward > 0:
        delta_w = self._learning_rate * (
            reward * ltp  # 正reward增强LTP
            - 0.1 * ltd   # 轻微LTD
        )
    elif reward < 0:
        delta_w = self._learning_rate * (
            0.1 * ltp     # 轻微LTP
            - abs(reward) * ltd  # 负reward增强LTD
        )
    else:
        # reward=0时，标准STDP
        delta_w = self._learning_rate * (ltp - ltd)

    # 权重约束
    self._weights += delta_w
    self._weights = np.clip(self._weights, self._w_min, self._w_max)

    # 记录权重变化历史
    self._weight_history.append(self._weights.copy())
```

**软STDP的优势**：

| 维度 | 传统硬STDP | Neo-MoFox软STDP |
|-----|-----------|----------------|
| **权重约束** | 无约束（可能爆炸） | 有界 [w_min, w_max] |
| **稳态调节** | 无（需要外部机制） | 内置稳态归一化 |
| **Reward调制** | 无（纯时序依赖） | Reward决定调整方向 |
| **计算效率** | 低（需精确时间戳） | 高（基于trace） |
| **在线学习友好** | 不友好（容易失衡） | 友好（自动调节） |

### 3.2 Trace机制

**Trace的定义**：记录神经元近期脉冲历史的衰减变量。

```python
# plugins/life_engine/snn/core.py::LIFNeuronGroup

class LIFNeuronGroup:
    """LIF神经元群。"""

    def __init__(self, n_neurons: int, tau_m: float = 20.0, tau_trace: float = 25.0):
        self._tau_trace = tau_trace  # Trace衰减常数（ms）

        # Trace变量（每个神经元一个trace）
        self._traces = np.zeros(n_neurons, dtype=np.float64)

    def step(self, input_spikes: np.ndarray) -> tuple[np.ndarray, np.ndarray]:
        """神经元步进更新。"""
        # 膜电位更新（LIF动力学）
        dV = (-self._voltages + input_spikes @ self._weights) / self._tau_m
        self._voltages += dV * dt

        # 脉冲产生（阈值检测）
        spikes = (self._voltages > self._threshold).astype(np.float64)
        self._voltages[spikes > 0] = self._reset_voltage  # 重置

        # Trace更新（衰减 + 脉冲增加）
        d_trace = (-self._traces + spikes) / self._tau_trace
        self._traces += d_trace * dt

        return spikes, self._voltages

    @property
    def traces(self) -> np.ndarray:
        """返回当前trace值。"""
        return self._traces
```

**Trace的物理意义**：

```
神经元在时间 t 的trace值 = Σ (最近脉冲 × 时间衰减)

例如：
神经元在过去 25ms 内产生了 3 个脉冲：
- 第 0ms：脉冲强度 1.0
- 第 10ms：脉冲强度 1.0
- 第 20ms：脉冲强度 1.0

当前 trace ≈ 1.0 * exp(-0/25) + 1.0 * exp(-10/25) + 1.0 * exp(-20/25)
           ≈ 1.0 + 0.67 + 0.45 ≈ 2.12

Trace反映了"近期的活跃程度"
```

**Trace在STDP中的作用**：

- **前trace高** → 前神经元近期活跃 → 可能导致LTP
- **后trace高** → 后神经元近期活跃 → 可能导致LTP
- **前高后低** → 前活跃后静默 → 可能导致LTD

### 3.3 稳态调节机制

**Homeostatic Scaling**：防止突触权重无限增长或衰减。

```python
# plugins/life_engine/snn/core.py::DriveCoreNetwork

def homeostatic_scaling(self, rate: float = 0.02):
    """稳态调节：权重归一化到基线范围。

    Args:
        rate: 调节速率（每次调整的幅度）
    """
    # 突触权重归一化
    for synapse in [self.syn_in_hid, self.syn_hid_out]:
        # 计算当前权重均值
        mean_weight = np.mean(synapse._weights)

        # 目标均值（基线）
        target_mean = 0.5

        # 向基线移动
        adjustment = rate * (target_mean - mean_weight)
        synapse._weights += adjustment

        # 重新约束到 [w_min, w_max]
        synapse._weights = np.clip(synapse._weights, synapse._w_min, synapse._w_max)
```

**稳态调节的时机**：

```python
# 每次NREM梦境回放后
if nrem_report.episodes_replayed > 0:
    snn.homeostatic_scaling(rate=0.02)
    nrem_report.homeostatic_applied = True
```

**稳态调节的意义**：

| 问题 | 无稳态调节 | 有稳态调节 |
|------|----------|----------|
| **权重爆炸** | 可能无限增长 | 自动归一化 |
| **权重消失** | 可能衰减到0 | 自动恢复 |
| **学习失衡** | LTP > LTD → 爆炸 | 动态平衡 |
| **网络稳定性** | 不稳定 | 长期稳定 |
