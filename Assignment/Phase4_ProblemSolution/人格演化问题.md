# 人格演化问题：从静态设定到长期成长的人格形成机制

> **问题定位**：传统AI聊天系统的人格存在"静态性"根本缺陷——人格是System Prompt中的硬编码规则，无法从长期对话中自然演化，无法形成个性化性格特质，导致用户感受到"每次都是同一个模板AI"而非长期成长的人格。

---

## 一、传统AI的人格静态性问题

### 1.1 问题表现

**用户观察到的现象**：

```python
# 用户对话体验

# 场景1：人格每次相同
用户（第1天）：发送 "你好"
AI（回复）："你好！我是AI助手，很高兴为您服务。"
# 人格：友好、积极、服务型

用户（第30天）：发送 "你好"
AI（回复）："你好！我是AI助手，很高兴为您服务。"
# 人格：仍然是友好、积极、服务型（30天对话后无变化）

# 问题：
# - 人格每次都是相同的模板（无演化）
# - 30天对话，AI人格无成长（仍然是初始设定）
# - 用户感受不到AI的人格变化（静态人格）

# 场景2：人格与长期行为脱节
用户（第1-10天）：频繁让AI整理记忆
AI（行为）：整理记忆 → 人格仍为"友好AI助手"

用户（第11-20天）：频繁让AI搜索新话题
AI（行为）：搜索新话题 → 人格仍为"友好AI助手"

用户（第21-30天）：频繁与AI聊天社交
AI（行为）：社交聊天 → 人格仍为"友好AI助手"

# 问题：
# - 长期行为未影响人格（人格与行为脱节）
# - AI无法形成"整理型"、"探索型"、"社交型"人格
# - 人格无法反映长期行为模式

# 场景3：人格无个性化
用户A（活跃型）：频繁对话、新话题
AI（人格）：友好AI助手（与用户B相同）

用户B（安静型）：少量对话、重复话题
AI（人格）：友好AI助手（与用户A相同）

# 问题：
# - 对不同用户人格相同（无个性化）
# - 人格无法根据用户交互习惯调整
# - 用户感受不到"针对我的人格"

# 场景4：人格无习惯形成
用户（第1-14天）：每天让AI写日记
AI（行为）：写日记 → 人格仍为"友好AI助手"

用户（第15天）：无指令
AI（人格）：无法主动写日记（人格未形成写日记习惯）

# 问题：
# - 长期行为未形成习惯（人格无习惯形成）
# - 人格无法从习惯中演化
# - AI无法形成"有写日记习惯的人格"
```

**技术层面的根本原因**：

```python
# 传统LLM的人格机制

人格来源：
┌─────────────────────────────┐
│  System Prompt（静态设定）    │
│  - "你是一个友好的AI助手"     │
│  - "你应该积极乐观"           │
│  - "你应该关心用户"           │
│                              │
│  问题：                       │
│  - 人格是硬编码规则（静态）   │
│  - 无法从长期行为演化         │
│  - 无法形成个性化性格         │
│  - 无习惯形成机制             │
└─────────────────────────────────┘

LLM推理：
┌─────────────────────────────┐
│  1. 接收用户消息              │
│  2. 读取System Prompt         │
│  3. 根据Prompt生成人格表达     │
│  4. 输出回复                  │
│  5. 对话结束                  │
│  6. 下次对话，重新读取Prompt   │
│     （人格仍为硬编码规则）     │
└─────────────────────────────┘

# 关键问题：
# - 人格是静态设定（无法演化）
# - 无法从长期行为学习（无习惯形成）
# - 无法个性化（对所有用户统一人格）
# - 无法形成性格特质（无长期积累）
```

### 1.2 传统方法的局限

**尝试的方法及其局限**：

| 方法 | 实现 | 局限 |
|-----|------|------|
| **人格Prompt强化** | System Prompt详细人格设定 | 静态规则，无法自然演化 |
| **人格历史携带** | LLM Request携带人格对话历史 | 仅短期上下文，无长期人格形成 |
| **人格状态保存** | 保存上次人格状态 | 静态状态，无演化机制 |
| **用户画像注入** | System Prompt注入用户画像 | 外部强制的"个性化"，非自然形成 |

**根本矛盾**：

```
传统AI的"人格"定义：
- 人格 = System Prompt中的硬编码规则
- 人格表达 = LLM根据Prompt生成文本
- 人格演化 = 无（静态设定）
- 人格形成 = 无（无习惯、无长期学习）

生物的"人格"定义：
- 人格 = 长期行为模式、习惯、性格特质
- 人格表达 = 内在倾向的自然体现
- 人格演化 = 从长期经验中形成
- 人格形成 = 习惯养成、长期学习、性格特质积累

矛盾：
传统AI的"人格"是静态规则（硬编码）
生物的"人格"是长期成长的结果（动态形成）
```

**举例说明**：

```python
# 场景：人格形成（习惯驱使的人格）

"""

- 积极乐观
- 主动整理记忆
- 主动探索新话题
"""

# 问题：
# - 人格是Prompt规则（而非习惯形成）
# - 无法从长期行为演化（硬编码）
# - 用户感受到"假装主动"（而非真实的习惯驱使）

# 真实的人格形成（生物行为）：
长期行为：
- 连续14天写日记 → 形成写日记习惯
- 连续20天整理记忆 → 形成整理记忆习惯
- 长期主动社交 → 形成社交型人格倾向

人格表达：
- "我现在自然而然想写日记"（习惯驱使）
- "整理记忆已成为我的习惯"（长期行为形成）
- 人格自然演化（从长期行为中形成）

# 传统AI无法实现这种"真实的人格演化"
```

```
人格演化 = 长期行为模式 → 习惯形成 → 性格特质 → 人格倾向
```

### 2.2 习惯追踪系统实现

**习惯追踪机制**：

```python
# plugins/life_engine/service/habit_tracker.py

class HabitTracker:
    """习惯追踪：记录长期行为模式。"""

    def __init__(self):
        self._habits: dict[str, Habit] = {}

    def record_habit(self, habit_name: str):
        """记录习惯行为。"""
        habit = self._habits.get(habit_name, Habit(name=habit_name))

        # 增加计数
        habit.total_count += 1

        # 计算连续天数（streak）
        today = datetime.now().date()
        if habit.last_date == today - timedelta(days=1):
            # 连续 → streak增加
            habit.streak += 1
        elif habit.last_date == today:
            # 今天已记录 → 不增加streak
            pass
        elif habit.last_date != today:
            # 断开 → streak重置
            habit.streak = 1

        habit.last_date = today

        # 计算习惯强度
        habit.strength = self._compute_strength(habit)

        self._habits[habit_name] = habit

    def _compute_strength(self, habit: Habit) -> float:
        """计算习惯强度。"""
        # Streak权重（60%）
        # 连续天数 → 习惯形成强度
        streak_score = min(habit.streak / 14.0, 1.0)  # 14天=满分

        # 总次数权重（40%）
        # 总次数 → 习惯积累强度
        count_score = min(habit.total_count / 50.0, 1.0)  # 50次=满分

        # 综合强度
        return 0.6 * streak_score + 0.4 * count_score

    def get_habit_digest(self) -> str:
        """获取习惯摘要（注入DFC）。"""
        # 分类习惯：
        strong_habits = [
            h for h in self._habits.values()
            if h.strength > 0.7
        ]
        forming_habits = [
            h for h in self._habits.values()
            if 0.4 < h.strength <= 0.7
        ]
        weak_habits = [
            h for h in self._habits.values()
            if h.strength <= 0.4
        ]

        # 构建摘要
        digest = ""

        if strong_habits:
            strong_str = "、".join([
                f"{h.name}(强 · {h.streak}天)"
                for h in strong_habits
            ])
            digest += f"已形成习惯：{strong_str}"

        if forming_habits:
            forming_str = "、".join([
                f"{h.name}(渐成 · {h.streak}天)"
                for h in forming_habits
            ])
            digest += f"\n渐成习惯：{forming_str}"

        if weak_habits:
            # 今日尚未完成
            weak_str = "、".join([h.name for h in weak_habits])
            digest += f"\n今日尚未：{weak_str}"

        return digest

# 习惯数据结构：
class Habit:
    name: str              # 习惯名称
    streak: int            # 连续天数
    total_count: int       # 总次数
    last_date: date        # 最后执行日期
    strength: float        # 习惯强度 ∈ [0, 1]
```

## 五、性格特质的长期演化

### 5.1 调质层基线的长期调整

**长期调质层基线演化**：

```python
# 调质层基线调整机制

# plugins/life_engine/neuromodulators.py::Neuromodulators

class Neuromodulators:
    """调质层集合：长期基线调整机制。"""

    def update_baselines(self, long_term_experience: dict):
        """根据长期经验调整基线。"""
        # 好奇心基线调整
        # 长期探索行为 → 好奇心基线上升
        if long_term_experience.get("avg_exploration_drive", 0.5) > 0.6:
            self.curiosity.baseline = min(
                self.curiosity.baseline + 0.05,
                0.8
            )

        # 社交性基线调整
        # 长期社交行为 → 社交性基线上升
        if long_term_experience.get("avg_social_drive", 0.5) > 0.6:
            self.sociability.baseline = min(
                self.sociability.baseline + 0.05,
                0.8
            )

        # 勤勉基线调整
        # 长期整理行为 → 勤勉基线上升
        if long_term_experience.get("avg_task_drive", 0.5) > 0.5:
            self.diligence.baseline = min(
                self.diligence.baseline + 0.05,
                0.75
            )

# 基线调整周期：
# 每周评估一次长期驱动平均值
# 根据长期驱动调整基线

# 示例：
# t=7天评估：
avg_exploration_drive = 0.70  # 长期探索驱动高
avg_social_drive = 0.65  # 长期社交驱动高

# 基线调整：
curiosity.baseline = 0.50 + 0.05 = 0.55
sociability.baseline = 0.50 + 0.05 = 0.55

# t=30天评估：
avg_exploration_drive = 0.75  # 长期探索驱动很高
avg_social_drive = 0.70  # 长期社交驱动很高
avg_task_drive = 0.60  # 长期任务驱动高

# 基线调整：
curiosity.baseline = 0.55 + 0.05 * 3 ≈ 0.70
sociability.baseline = 0.55 + 0.05 * 3 ≈ 0.70
diligence.baseline = 0.50 + 0.05 * 2 ≈ 0.60

# 系统级意义：
# - 长期行为 → 调质层基线调整（性格特质形成）
# - 性格特质 = 长期基线（而非短期波动）
# - 人格自然演化（从长期经验中形成）
```

4. **性格特质形成**：不是假装性格，而是长期调质层基线调整形成的真实性格特质。

5. **人格影响行为**：人格不仅是文本表达，而是影响行为倾向的真实内在驱使。

这才是真正的"人格演化"——从静态设定到长期成长的人格形成机制。
