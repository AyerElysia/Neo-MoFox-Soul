### 2.2 SNN在线学习详解

**每次心跳的学习机制**：

```python
# plugins/life_engine/snn/core.py::DriveCoreNetwork

def step(self, input_vec: np.ndarray, reward: float = 0.0) -> np.ndarray:
    """真实输入步（带学习的完整更新）。"""
    # 1. 输入特征提取（从事件流）
    input_scaled = input_vec * self._input_gain

    # 2. 脉冲传播（前向）
    current_hidden = self.syn_in_hid.forward(input_scaled)
    spikes_hidden = self.hidden.step(current_hidden)

    hidden_spike_signal = spikes_hidden.astype(np.float64) * self._hidden_spike_gain
    current_output = self.syn_hid_out.forward(hidden_spike_signal)
    spikes_output = self.output.step(current_output)

    # 3. STDP学习（权重调整）
    # - 根据reward信号调整学习方向
    # - reward > 0 → 强化成功路径（LTP）
    # - reward < 0 → 抑制失败路径（LTD）

    hidden_v = self.hidden.get_state()
    output_v = self.output.get_state()

    # 软STDP（基于trace的连续学习）
    soft_hidden = _sigmoid(hidden_v, center=self.hidden.threshold * 0.5)
    soft_output = _sigmoid(output_v, center=self.output.threshold * 0.5)
    input_activity = np.abs(input_vec) / max(np.max(np.abs(input_vec)), 0.01)

    # 突触权重更新（输入层→隐藏层）
    self.syn_in_hid.update_soft(input_activity, soft_hidden, reward=reward)

    # 突触权重更新（隐藏层→输出层）
    self.syn_hid_out.update_soft(soft_hidden, soft_output, reward=reward)

    # 4. 驱动输出
    raw_output = self.output.get_state()
    self._output_ema = (1 - self._ema_alpha) * self._output_ema + self._ema_alpha * raw_output

    return self._output_ema.copy()
```

**STDP学习规则详解**：

```python
# plugins/life_engine/snn/core.py::STDPSynapse

def update_soft(
    self,
    pre_activity: np.ndarray,
    post_activity: np.ndarray,
    reward: float = 0.0,
):
    """软STDP更新：用连续活跃度而非二值spike。"""
    # Trace更新（记录脉冲历史）
    self.trace_pre = self.trace_pre * self.trace_decay + pre_activity
    self.trace_post = self.trace_post * self.trace_decay + post_activity

    # Reward调制
    reward_factor = 1.0 + np.clip(reward, -1.0, 1.0)

    # 计算权重变化
    pre_strength = float(np.sum(pre_activity))
    post_strength = float(np.sum(post_activity))

    # LTP（长时程增强）
    if post_strength > 0.05:
        dw_plus = self.lr_plus * np.outer(post_activity, self.trace_pre)
        self.W += dw_plus * max(reward_factor, 0.1)

    # LTD（长时程抑制）
    if pre_strength > 0.05:
        dw_minus = -self.lr_minus * np.outer(self.trace_post, pre_activity)
        self.W += dw_minus * max(2.0 - reward_factor, 0.1)

    # 权重约束
    np.clip(self.W, self.w_min, self.w_max, out=self.W)
```

### 2.3 Reward信号来源

**Reward计算机制**：

```python
# plugins/life_engine/snn/bridge.py::compute_reward

def compute_reward(
    tool_event_count: int = 0,
    tool_success_count: int = 0,
    tool_fail_count: int = 0,
    idle_heartbeat_count: int = 0,
    had_text_reply: bool = True,
) -> float:
    """从心跳结果计算reward信号。"""
    reward = 0.0

    # 工具调用奖励
    tool_calls = tool_event_count // 2 if tool_event_count > 0 else 0

    if tool_calls > 0:
        reward += 0.3  # 有工具调用 → 基础奖励

    # 工具成功奖励（强化成功路径）
    if tool_success_count > 0:
        reward += min(tool_success_count * 0.15, 0.4)

    # 无工具调用惩罚（抑制空转）
    if tool_calls == 0:
        reward -= 0.2

    # 工具失败惩罚（抑制失败路径）
    if tool_fail_count > 0:
        reward -= min(tool_fail_count * 0.2, 0.4)

    # 空转惩罚（抑制长时间idle）
    if idle_heartbeat_count >= 5:
        reward -= 0.3
    elif idle_heartbeat_count >= 2:
        reward -= 0.15

    # 裁剪到 [-1, 1]
    return float(np.clip(reward, -1.0, 1.0))
```

**Reward信号的系统意义**：

| Reward来源 | Reward值 | 学习效果 |
|-----------|---------|---------|
| **工具成功** | +0.3 ~ +0.7 | 强化成功路径（LTP） |
| **工具失败** | -0.2 ~ -0.4 | 抑制失败路径（LTD） |
| **无工具调用** | -0.2 | 抑制空转路径 |
| **空转过多** | -0.15 ~ -0.3 | 抑制长时间idle |
| **社交成功** | +0.2 | 强化社交驱动 |

**Hebbian学习详解**：

```python
# plugins/life_engine/memory/edges.py::reinforce_coactivated

async def reinforce_coactivated(
    db: sqlite3.Connection,
    node_ids: list[str],
    learning_rate: float = 0.1,
):
    """Hebbian学习：强化共激活节点之间的连接。"""
    if len(node_ids) < 2:
        return

    # 获取节点激活强度
    activations = {}
    for node_id in node_ids:
        cursor.execute("SELECT activation_strength FROM memory_nodes WHERE node_id = ?", (node_id,))
        row = cursor.fetchone()
        if row:
            activations[node_id] = float(row[0])

    # 遍历所有共激活节点对
    for i, source_id in enumerate(node_ids):
        for target_id in node_ids[i+1:]:
            # 计算共激活强度
            source_activation = activations.get(source_id, 0.5)
            target_activation = activations.get(target_id, 0.5)
            coactivation_strength = source_activation * target_activation

            # 强化连接
            await create_or_update_edge(
                source_id=source_id,
                target_id=target_id,
                edge_type=EdgeType.ASSOCIATES,
                reason="Hebbian共激活强化",
                strength_change=learning_rate * coactivation_strength,
            )
```

## 六、技术创新总结

### 6.1 在线学习的核心技术

| 创新点 | 技术实现 | 系统意义 |
|-------|---------|---------|
| **SNN在线学习** | 每心跳STDP调整 | 驱动产生演化 |
| **Reward信号** | 事件结果计算 | 学习方向引导 |
| **Hebbian学习** | 检索时共激活强化 | 记忆网络建立 |
| **记忆演化** | 连接建立+剪枝 | 记忆结构优化 |
| **习惯追踪** | streak+strength | 行为模式形成 |
| **个性化适应** | 根据用户事件流 | 差异化人格 |
| **错误学习** | 负reward → LTD | 从失败中改进 |
| **稳态调节** | homeostatic scaling | 长期稳定学习 |

### 6.2 系统级意义

**在线学习的核心价值**：

1. **持续成长能力**：不是能力冻结的模型，而是持续演化的生命系统。

2. **个性化适应**：不是对所有用户统一模型，而是根据每个用户的事件流差异化。

3. **从错误中学习**：不是重复犯错，而是从失败中调整驱动产生。

4. **经验积累**：不是每次对话独立，而是长期经验记录在突触权重和记忆网络。

5. **能力演化**：不是静态能力，而是随时间持续成长（驱动产生、记忆结构、人格倾向）。

这才是真正的"在线学习"——从冻结模型到持续成长的生命系统。
