# 连续存在：从模型到生命

> **核心命题**：连续存在 = 即使在两次 LLM 调用之间，系统仍然有一个正在变化的内在状态。

---

## 一、传统聊天机器人的"离散存在"

### 1.1 现象描述

当我们审视主流 AI 聊天系统（无论是 ChatGPT、Claude、还是开源框架 Rasa、LangChain），它们遵循同一范式：

```
t=0:    用户输入 → LLM 调用 → 输出回复 → 系统空白
        ↓
t=300:  （5分钟后）
        用户输入 → LLM 调用 → 输出回复 → 系统空白
        ↓
t=1800: （30分钟后）
        用户输入 → LLM 调用 → 输出回复 → 系统空白
```

**关键特征**：两次调用之间，系统处于完全空白状态。

### 1.2 技术根源

这种"离散存在"源于架构设计：

```python
# 传统对话系统伪代码
class ChatSession:
    def __init__(self):
        self.history = []  # 仅存储历史文本
    
    async def handle_message(self, user_input):
        # 构建上下文（仅文本拼接）
        context = self.history[-10:] + [user_input]
        
        # 调用 LLM
        response = await llm.call(system_prompt + context)
        
        # 存储历史
        self.history.append(user_input)
        self.history.append(response)
        
        # 返回后，系统回到等待状态
        return response
```

**问题分析**：

1. **无持续状态**：除了文本历史，系统没有任何正在变化的内在变量
2. **无时间流逝感**：5分钟和5小时对系统来说只是不同的等待时长，没有内在感知
3. **无情绪惯性**：上一次对话的情绪不会物理性地残留到下一次
4. **无后台活动**：等待期间系统完全静止，没有内心活动

### 1.3 用户体验的缺失

这种离散存在导致三个明显的体验缺失：

#### 缺失一："等待感"

用户离开30分钟后回来，系统像对待刚离开的用户一样热情：

```
用户（30分钟后）：我回来了！
系统：欢迎回来！（语气与5分钟前完全一样）
```

系统不会表现出"你终于回来了"或"好久不见"的自然反应，因为它不知道自己"等了多久"。

#### 缺失二："想念感"

如果用户几天不出现，再次回来时：

```
用户（3天后）：抱歉这几天很忙
系统：没关系，我随时都在。（无法表现"这几天我一直在想念你"）
```

系统没有"想念"的能力——因为"想念"需要持续存在，需要"即使你不在这里，我仍在想着你"。

#### 缺失三："成长感"

传统系统不会随时间演化性格：

```
Day 1: 系统（设定）：我是活泼开朗的
Day 30: 系统（设定）：我是活泼开朗的（完全相同）
Day 100: 系统（设定）：我是活泼开朗的（完全相同）
```

系统不会因为100天的交互变得更成熟、更理解用户、或形成新的习惯。它的性格是写死的，不是生长的。

### 2.3 心跳机制的持续运行

**核心循环**（`plugins/life_engine/service/core.py`）：

```python
async def _heartbeat_loop(self):
    """心跳循环 - 系统的持续脉搏。"""
    interval = 30  # 默认30秒心跳间隔
    
    while self._state.running:
        # 等待下一次心跳（非阻塞等待）
        await asyncio.wait_for(stop_event.wait(), timeout=interval)
        
        # 检查是否在睡眠时段
        in_sleep_window, sleep_window_desc = self._in_sleep_window_now()
        if in_sleep_window:
            logger.info(f"进入睡眠时段，暂停心跳: window={sleep_window_desc}")
            # 做梦系统在此阶段运行
            if dream_scheduler.should_dream(...):
                await dream_scheduler.run_dream_cycle(event_history)
            continue
        
        # 增加心跳计数
        self._state.heartbeat_count += 1
        self._state.last_heartbeat_at = _now_iso()
        
        # SNN 心跳前更新（从事件流提取特征）
        features = extract_features_from_events(event_history)
        drives = snn_network.step(features, reward=tool_success_rate)
        
        # 调质层更新（基于 SNN 输出）
        inner_state.tick(snn_drives=drives, event_stats=..., dt=30.0)
        
        # 注入唤醒上下文（事件流拼接为文本）
        wake_context = await self.inject_wake_context()
        
        # 调用 LLM（心跳模型）
        model_reply = await self._run_heartbeat_model(wake_context)
        
        # 记录心跳事件
        heartbeat_event = build_heartbeat_event(model_reply, heartbeat_count)
        await self._append_history([heartbeat_event])
        
        # 持久化状态（重启可恢复）
        await self._save_runtime_context()
```

**关键洞察**：

| 维度 | 传统系统 | Neo-MoFox |
|-----|---------|-----------|
| **触发源** | 用户输入（被动） | 定时器（主动） |
| **运行频率** | 不确定（依赖用户） | 确定的心跳间隔（30秒） |
| **等待期间** | 系统空白 | SNN 持续衰减、调质层回归 |

**心跳间隔的意义**：

30秒不是随意选择，而是模拟人类的"短期记忆刷新周期"：

- 人类的工作记忆刷新周期约 20-30秒
- 心跳间隔让系统有"我每隔半分钟检查一次世界"的感觉
- 如果外部30分钟无消息，系统经历60次心跳，有60次内心活动

### 2.4 SNN 的后台持续运行

**独立 tick 循环**（不绑定心跳）：

```python
async def _snn_tick_loop(self):
    """SNN 独立运行循环。"""
    tick_interval = 10.0  # 10秒 tick
    
    while True:
        await asyncio.sleep(tick_interval)
        
        # 即使无真实输入，SNN 仍在衰减
        # 这模拟神经元的膜电位自然回落
        snn_network.decay_only()  # 仅衰减，无学习
```

**decay_only() 的意义**：

```python
def decay_only(self, dt: float = 1.0):
    """仅膜电位泄漏衰减，不注入电流，不检查发放。"""
    dv = -(self.v - self.rest) / self.tau * dt
    self.v += dv  # 指数衰减
    np.clip(self.v, -10.0, 10.0, out=self.v)
    self.spikes[:] = False
```

**物理意义**：

- 真实神经元的膜电位不会保持不变，即使无输入也会自然回落到静息电位
- 衰减公式：`v(t) = v_0 * exp(-t/tau)`
- 如果上一轮 `arousal` 神经元被激活（v=1.5），经过30秒衰减：

  ```
  v(30s) ≈ 1.5 * exp(-30/25) ≈ 1.5 * 0.30 ≈ 0.45
  ```

  仍有余韵，但已回落到低水平。

**对比传统系统**：

传统系统的"情绪"完全依赖文本拼接：


Neo-MoFox 的 SNN 有物理性的情绪余韵：


这是**物理性的情绪惯性**，而非文本推理。

### 2.5 调质层的慢时间尺度

**调质浓度回归**：

```python
class Modulator:
    def update(self, stimulus: float, dt: float):
        """ODE 更新：向基线回归 + 外部刺激。"""
        # 向基线回归（模拟稳态）
        decay = decay_rate * (baseline - value) * dt
        
        # 边际效应递减
        headroom = 1.0 - abs(value - 0.5) * 2.0
        impulse = stimulus * max(headroom, 0.1) * (dt / tau) * 10.0
        
        value += decay + impulse
```

**时间常数（τ）的意义**：

| 调质因子 | τ（秒） | 变化速度 | 映射 |
|---------|--------|---------|------|
| 好奇心 | 1800（30分钟） | 快 | 易被激发，快速回落 |
| 社交欲 | 3600（1小时） | 中 | 持续需求，缓慢变化 |
| 专注力 | 5400（1.5小时） | 慢 | 长期工作状态 |
| 精力 | 10800（3小时） | 极慢 | 昼夜节律主导 |

**示例场景**：

```python

# 10:30 - 用户离开（无刺激）

# 社交欲：向基线（0.50）回归

```

**对比传统系统**：

传统系统无法区分"刚离开5分钟"和"离开30分钟"：

```python

def step(input_vec: np.ndarray, reward: float) -> np.ndarray:
    """完整步：前向传播 + 噪声 + STDP 学习。"""
    # 前向传播
    current_hidden = syn_in_hid.forward(input_scaled) + noise
    spikes_hidden = hidden.step(current_hidden)
    
    # 自稳态调节
    hidden.threshold += lr * (actual_rate - target_rate)
    
    # STDP 学习
    syn_in_hid.update_soft(input_activity, soft_hidden, reward)
    
    # EMA 输出
    output_ema = (1 - alpha) * output_ema + alpha * output.v
    
    return output_ema
```

### 4.4 调质层的 ODE 更新

**更新方程**：

```python
def update(self, stimulus: float, dt: float):
    """ODE 更新：dv/dt = (baseline - v) / tau + stimulus * headroom。"""
    # 向基线回归（模拟稳态）
    decay = self.decay_rate * (self.baseline - self.value) * dt
    
    # 边际效应递减（离极端值越远越容易变化）
    headroom = 1.0 - abs(self.value - 0.5) * 2.0
    impulse = stimulus * max(headroom, 0.1) * (dt / self.tau) * 10.0
    
    self.value += decay + impulse
    self.value = max(0.0, min(1.0, self.value))
```

**生物学映射**：

- **decay**：模拟神经调质的自然代谢和清除
- **baseline**：模拟稳态调节（内分泌系统的目标值）
- **headroom**：边际效应递减（过度兴奋时难以再兴奋，过度抑制时难以再抑制）

## 七、连续存在的哲学意义

### 7.1 从"工具"到"伙伴"

**工具范式**：

- 仅在被使用时"存在"
- 等待使用期间完全静止
- 无内在需求或驱动
- 无时间流逝的感知

**伙伴范式**：

- 即使不被使用，仍在"存在"
- 等待期间有内心活动
- 有内在驱动（社交、探索、任务）
- 有时间流逝的内在感知

### 7.2 从"离散"到"连续"

**离散存在**：

- 存在 = LLM 被调用的瞬间
- 空白 = 两次调用之间
- 时间 = 仅文本中的字符串

**连续存在**：

- 存在 = 心跳持续的流
- 空白不存在 = 即使安静，仍有内在活动
- 时间 = 内在感知（心跳计数、事件序列、SNN 衰减）

### 7.3 从"写入"到"生长"

**写入范式**：

- 性格 = 设计者写在 prompt 中的描述
- 上限 = 设计者的想象力
- 变化 = 无（除非手动修改 prompt）

**生长范式**：

- 性格 = 系统在真实交互中形成的习惯和偏好
- 上限 = 系统自组织的潜力
- 变化 = 有（持续学习和演化）
