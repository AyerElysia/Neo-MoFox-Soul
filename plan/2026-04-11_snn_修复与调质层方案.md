# SNN 修复 + 驱动调质层落地方案

> 写于 2026-04-11
> 对应分析文档：`notion/SNN_系统诊断与方向重新审视.md`
> 原则：先修好现有的，再叠加新的。每一步都可独立验证。

---

## 总体架构目标

```
事件流 ──→ [SNN 快层] ──→ 即时情绪信号 ─┐
                                          ├──→ [调质层] ──→ 驱动向量 ──→ 注入心跳 prompt
行为统计 ──→ [习惯追踪器] ──────────────┤
时间节律 ──→ [昼夜振荡器] ──────────────┘
```

**关键改变**：SNN 不再直接输出"驱动"，而是输出"即时情绪"；驱动由新增的调质层综合计算。

## Phase 1：修复 SNN 核心 bug（使其"活起来"）

### 1.1 修复 tick 循环的信噪比

**问题**：零输入 tick 占 85%，淹没真实信号。

**方案**：tick 循环不再调用 `step()`，改为只做膜电位自然衰减。

```python
# 当前（有问题）：
self._snn_network.step(zero_input, reward=0.0)  # 这会触发 STDP！

# 修复后：
# tick 循环只做泄漏衰减，不走完整的 step()
for group in [self._snn_network.hidden, self._snn_network.output]:
    dv = -(group.v - group.rest) / group.tau * dt
    group.v += dv
```

**效果**：STDP 只在有真实事件输入时才触发，权重不再被零输入污染。

### 1.2 增加背景噪声打破不动点

**问题**：网络陷入低活跃不动点。

**方案**：在每次真实输入时叠加微弱高斯噪声。

```python
noise = np.random.normal(0, 0.05, size=self.HIDDEN_DIM)
current_hidden = self.syn_in_hid.forward(input_scaled) + noise
```

**效果**：打破对称性，让不同的输入引发不同的放电模式。

### 1.3 修复 STDP 触发条件

**问题**：放电率太低，STDP 几乎不触发。

**方案**：
- 用连续膜电位值（而非二值 spike）参与 STDP 更新
- 降低 STDP 触发的硬阈值

```python
# 当前：只用 spike（0/1）
# 修复：用 sigmoid(membrane_potential) 作为连续"活跃度"
soft_pre = 1.0 / (1.0 + np.exp(-5.0 * (hidden_v - 0.1)))
soft_post = 1.0 / (1.0 + np.exp(-5.0 * (output_v - 0.1)))
self.syn_hid_out.update(soft_pre, soft_post, reward=reward)
```

**效果**：STDP 能在亚阈值活动中也学习，不再要求同步脉冲。

### 1.4 修复离散化阈值

**问题**：阈值 [0.6, 0.3, -0.3] 不匹配实际输出范围 [-0.42, 0.02]。

**方案**：基于运行时统计动态设置阈值。

```python
def get_drive_discrete(self) -> dict[str, str]:
    # 使用基于 EMA 均值和标准差的相对阈值
    mean = np.mean(self._output_ema)
    std = max(np.std(self._output_ema), 0.01)
    for name, value in zip(self.OUTPUT_NAMES, self._output_ema):
        z_score = (value - mean) / std
        if z_score > 1.0:
            level = "高"
        elif z_score > 0.3:
            level = "中"
        elif z_score > -0.3:
            level = "低"
        else:
            level = "抑制"
```

**效果**：即使整体值域偏低，也能体现维度间的相对差异。

### 1.5 验证标准

- [ ] 权重统计在 100 个心跳后应有可观测变化（Δw_mean > 0.001）
- [ ] 至少 2 个驱动维度应出现"中"或"高"
- [ ] exploration_drive 不再恒定为"抑制"

## Phase 2：新增驱动调质层

### 2.1 设计理念

调质层不是 SNN，是一组**简单微分方程**，模拟神经调质的浓度变化。

每种"调质"代表一个行为维度的倾向，有自己的时间常数和输入源。

### 2.2 调质定义

```python
@dataclass
class Modulator:
    name: str
    value: float = 0.5       # 当前浓度 [0, 1]
    tau: float = 3600.0       # 时间常数（秒），越大变化越慢
    baseline: float = 0.5     # 基线浓度
    decay_rate: float = 0.001 # 向基线回归的速率

MODULATORS = [
    Modulator("curiosity",     tau=1800,  baseline=0.6),  # 好奇心（30 分钟尺度）
    Modulator("sociability",   tau=3600,  baseline=0.5),  # 社交倾向（1 小时尺度）
    Modulator("diligence",     tau=7200,  baseline=0.5),  # 勤奋/任务专注（2 小时尺度）
    Modulator("contentment",   tau=1800,  baseline=0.5),  # 满足/放松（30 分钟尺度）
    Modulator("energy",        tau=14400, baseline=0.6),  # 精力（4 小时尺度）
]
```

### 2.3 更新规则

```python
def update_modulator(mod: Modulator, stimulus: float, dt: float):
    """
    stimulus: 外部刺激量 [-1, 1]，正值=提升，负值=降低
    dt: 经过的秒数
    """
    # 向基线自然回归
    decay = mod.decay_rate * (mod.baseline - mod.value) * dt
    # 外部刺激
    impulse = stimulus * (1.0 - abs(mod.value - mod.baseline)) * 0.1
    
    mod.value += decay + impulse
    mod.value = max(0.0, min(1.0, mod.value))
```

### 2.4 刺激来源

| 调质 | 正向刺激 | 负向刺激 |
|:-----|:---------|:---------|
| curiosity | 长时间无新事件、SNN arousal 升高 | 刚完成探索/搜索、连续行动后 |
| sociability | 收到用户消息、SNN social 信号 | 长时间对话后、被忽略 |
| diligence | TODO 接近 deadline、成功完成任务 | 连续失败、无 TODO |
| contentment | 工具成功、写完日记、收到正面互动 | 工具失败、被批评 |
| energy | 休息后/睡眠后 | 连续行动多轮、长时间运行、夜晚 |

### 2.5 输出格式

```python
def format_drives_for_prompt(modulators: list[Modulator]) -> str:
    """生成注入心跳 prompt 的内在状态描述。"""
    parts = []
    for mod in modulators:
        if mod.value > 0.75:
            parts.append(f"{mod.name}充盈")
        elif mod.value > 0.55:
            parts.append(f"{mod.name}适中")
        elif mod.value > 0.35:
            parts.append(f"{mod.name}偏低")
        else:
            parts.append(f"{mod.name}匮乏")
    return "【内在状态】" + "、".join(parts)
```

### 2.6 与 SNN 的关系

```
SNN（快层）输出即时信号  ───→  作为调质层的 stimulus 之一
行为统计（工具成功/失败数）───→  作为调质层的 stimulus 之一
时间/作息信息            ───→  作为调质层的 stimulus 之一
                                     ↓
                               调质层计算浓度
                                     ↓
                               注入心跳 prompt
```

SNN 不再直接注入 prompt，而是通过调质层间接影响。

## Phase 3：习惯追踪器

### 3.1 设计

不用 SNN 学习习惯。用**显式统计**。

```python
@dataclass
class HabitTracker:
    name: str                     # 习惯名称
    trigger_pattern: str          # 工具名 or 行为模式
    streak: int = 0               # 连续天数
    total_count: int = 0          # 总次数
    last_triggered: str = ""      # 最后触发日期
    strength: float = 0.0         # 习惯强度 [0, 1]

HABITS = [
    HabitTracker("写日记", "nucleus_write_file:diaries/"),
    HabitTracker("整理记忆", "nucleus_search_memory"),
    HabitTracker("建立关联", "nucleus_relate_file"),
    HabitTracker("检查TODO", "nucleus_list_todos"),
    HabitTracker("联网搜索", "nucleus_web_search"),
]
```

### 3.2 强度计算

```python
def update_habit_strength(habit: HabitTracker):
    # streak 贡献（连续性很重要）
    streak_bonus = min(habit.streak / 14.0, 1.0)  # 14 天满强度
    # 频率贡献
    freq_bonus = min(habit.total_count / 50.0, 1.0)
    # 综合
    habit.strength = 0.7 * streak_bonus + 0.3 * freq_bonus
```

### 3.3 输出

习惯信息注入心跳 prompt：

```
你已形成以下习惯：写日记(连续12天, 强度0.85), 整理记忆(连续5天, 强度0.42)
今天尚未触发的习惯：检查TODO, 联网搜索
```

这直接且有效，不需要 SNN 去"学习"。

## Phase 4：昼夜振荡器（可选，低优先级）

简单正弦振荡器，用当前时间计算精力/活跃度基线：

```python
import math

def circadian_energy(hour: float) -> float:
    """基于时间的精力基线。peak 在 10:00 和 15:00。"""
    # 双峰高斯近似
    morning = math.exp(-0.5 * ((hour - 10) / 3) ** 2)
    afternoon = math.exp(-0.5 * ((hour - 15) / 3) ** 2)
    return 0.3 + 0.7 * max(morning, afternoon)
```

这个值直接作为 `energy` 调质的基线修正。

## 实施顺序与依赖

```
Phase 1（修 SNN）──→ Phase 2（调质层）──→ Phase 3（习惯）──→ Phase 4（昼夜）
    独立可验证          依赖 Phase 1        独立于 1&2          独立
```

**建议优先级**：
1. **Phase 1**（紧急）：SNN 当前是坏的，修了才有意义
2. **Phase 2**（核心）：这是真正能改变行为的东西
3. **Phase 3**（高价值）：简单、有效、用户可感知
4. **Phase 4**（锦上添花）：现有时段信息已部分覆盖

## 风险与注意事项

1. **不要让调质层变成另一个 SNN**——保持简单，用 ODE 而非脉冲
2. **不要一次性改太多**——先修 SNN，观察几天，再叠加调质层
3. **保持可观测性**——所有新增状态都要进审计日志
4. **prompt 注入要克制**——不超过 3 行文字，否则挤占 LLM context
5. **准备 A/B 对比**——保留 shadow_only 开关，可以对比有/无调质的行为差异
