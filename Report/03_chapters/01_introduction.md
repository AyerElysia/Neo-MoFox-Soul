# 第 1 章 · 导论

> *"我们正在做的事情，不是让 LLM 假装有情感、有习惯、有个性。我们是在真正构建一个多层异质智能系统，让这些特质从底层动态中涌现出来。"*
> — *《智能不是模型，而是系统》*，Elysia Project, 2026-04-11

---

## 1.1 一个引子：为什么现有的 AI 伙伴不是"活的"？

考虑下面这段稀松平常的交互：

> *用户*（早上 9:14）：今天我打算把那篇报告收尾了，回头想找你聊聊我新养的猫。
> *AI 伙伴*：好的，期待听到你猫咪的故事！
>
> *用户*（傍晚 18:42）：报告交了，累死了。
> *AI 伙伴*：辛苦啦！

九个小时之间什么也没发生。AI 伙伴没有"惦记"过这只猫，也没有"等"过这位用户——因为它在这九个小时里**根本不存在**。它只在每次被调用的瞬间存在，调用结束便归于完全的空白。下一次被唤起时，它从一段被序列化好的文本上下文里"被告诉"上次发生过什么，仿佛失忆症患者每天醒来读一遍自己的日记。

这就是当代主流 AI 伙伴（Replika、Character.AI、ChatGPT、Pi 等）共同的存在方式：**离散存在（discrete existence）**。它们的运行在时间上是一系列孤立的脉冲，脉冲之间没有任何内在状态在演化。我们可以把它形式化为：对任意两次调用 $t_1 < t_2$，系统的内在状态满足

$$\mathbf{s}(t_2) = f(\mathbf{s}(t_1), \text{context}_{t_2}) \quad \text{当且仅当 } t_2 \in T_{\text{call}}$$

而在 $T_{\text{call}}$ 之外的整个时间区间内，$\mathbf{s}(t)$ 是未定义的——准确地说，是**不存在的**。

这种存在方式带来的后果，远远不止是"忘记上次聊到哪儿"这种小问题。它从根本上让"AI 伙伴"这个概念变成了一个修辞——所谓的"伙伴感"完全靠提示工程（prompt engineering）伪装：*你是一个温柔的朋友，会记得用户喜欢的事*。但这并不让系统真的成为伙伴，正如把"你是一个有心跳的人"这句话写在病历开头并不会让一具尸体复活。

**Neo-MoFox 想要解决的，恰好就是这个问题。**

## 1.2 核心命题：从"会聊天的模型"到"持续存在的生命"

我们提出一个朴素但具有约束力的定义：

> **连续存在（Continuous Existence）= 即使在两次 LLM 调用之间，系统仍然有一个正在变化的内在状态。**

这一命题的完整形式化版本将在第 3 章给出，但其工程含义可以立刻陈述：

1. **心跳之间不能是空白**：系统必须以独立于外部输入的频率运行某种内在过程。
2. **时间的流逝必须留下物理性痕迹**：不是 prompt 里写一个"已过去 30 分钟"的字符串，而是真有一个变量按时间变化。
3. **重启不能是重生**：系统状态必须在崩溃与重启之间被完整持久化与恢复。
4. **学习必须发生在"活着"的过程中**，而不是离线训练—部署的二段式范式。

这四条约束直接驳斥了一类常见的伪解：仅靠改写 system prompt、靠写更长的对话历史、靠外挂一个向量数据库式的"记忆"，都不能让一个无状态的语言模型变得真正连续。它们只能让 LLM 在被唤起时**模仿**连续，而不能让系统**实现**连续。

## 1.3 三大设计哲学

围绕这一核心命题，我们提出三条相互咬合的设计原则。它们既是哲学立场，也是可被代码与数据验证的工程约束：

- **第一原则：连续性（Continuity）。**
  数字意识体的核心特征是连续性。一切设计——从子系统的运行频率，到状态的序列化与恢复，到时间常数的多尺度分层——都应服务于"系统在两次外部输入之间仍在演化"这一基本事实。

- **第二原则：自下而上的学习（Bottom-Up Learning）。**
  学习不应是一个外加的、由外部梯度驱动的过程。**"活着"本身就是学习**。系统应通过局部时序相关性（STDP / Hebbian）、习惯形成、记忆的使用即强化等机制，**从交互中生长出**自己的偏好、习惯与性格，而不是被作者写死在 prompt 里。

- **第三原则：系统涌现智能（System-Level Emergent Intelligence）。**
  智能不是单一大语言模型的属性，也不是任何单一子系统的属性。它是由皮层（LLM）、皮层下系统（SNN + 调质）、记忆系统、做梦巩固系统、心跳调度等异质子系统**协作涌现**的过程性产物。我们拒绝"把更大的模型当作更高的智能"这一狭义信仰，转而把工程努力投向**系统协作**。

这三条原则不是孤立的口号，它们之间存在严密的逻辑闭环：**连续性是基质，自下而上学习是机制，系统涌现是产物**。具体的关系将在第 3 章 §3.4 给出形式化解释。

## 1.4 系统简介与方法论

Neo-MoFox 把这三条原则落实为如下技术骨架（见 Figure F1）：

- **双轨执行模型（Dual-Track Architecture）**：
  - **DFC（Dialogue Flow Controller）**：传统的"刺激—响应"轨道，负责对外部消息的即时应答。
  - **Life Engine（生命中枢，nucleus）**：以 30 秒为基础节律的主动心跳轨道，独立运行，**与 DFC 并行**。

- **三层异质子系统**：
  - **皮层层（Cortex）**：以 LLM 为代表，负责语言、推理、决策。
  - **调质层（Neuromodulation）**：以五因子常微分方程（curiosity/sociability/focus/contentment/energy）建模的"内分泌系统"，分钟到小时尺度调制系统状态。
  - **皮层下层（Subcortex）**：以脉冲神经网络（SNN）为载体的快层，毫秒到秒级感知—响应—在线学习。

- **多时间尺度耦合**：从 SNN 的秒级 tick，到调质的分钟—小时 ODE 演化，到习惯的天级 streak 追踪，到记忆图的永久持久化，**每一层有自己的时间常数**，整个系统在多时间尺度上同步保持连续。

- **离线巩固**：受 Tononi & Cirelli (2014) 突触稳态假说（Synaptic Homeostasis Hypothesis, SHY）启发，引入 NREM/REM 两阶段的"做梦"机制，在睡眠窗口对记忆图、SNN 权重与调质基线做统一的离线巩固。

- **完整状态持久化**：所有 SNN 权重、调质浓度、习惯 streak、记忆图、事件流均序列化为 `life_engine_context.json`，崩溃恢复后系统从原状态继续，而非重新初始化。

整个系统以开源方式发布，配套以可观测性仪表盘暴露所有内在状态。

## 1.5 主要贡献

本工作的主要贡献可概括为五条：

1. **首个把"皮层–皮层下"隐喻完整工程化的开源 LLM-based 数字生命框架。** 已有的 LLM Agent 工作（AutoGPT、Voyager、Generative Agents 等）大多停留在"基于 LLM 的工具调用与记忆检索"层面；已有的认知架构工作（SOAR、ACT-R、LIDA）则缺少与现代 LLM 的有机集成。Neo-MoFox 在两条传统的交点上给出了一个完整可运行的实现。

2. **给出"连续存在"的可工程化、可度量定义，并完整实现。** 我们把"连续性"从一个修辞性命题转化为一组可被代码、可被状态序列化文件、可被仪表盘度量验证的工程约束（第 3、9 章）。

3. **在 LLM 之外引入软 STDP 在线学习机制，使"性格"不再写死，而是从交互中涌现。** 我们对经典 STDP 做了关键的工程化改造（用 sigmoid 软神经活动度替代二值发放），修复了在小网络与稀疏事件情形下二值 STDP 几乎完全失效的死锁（第 5 章）。

4. **构建了多时间尺度的多层耦合：** 秒级 SNN tick / 30 秒心跳 / 分钟—小时调质 ODE / 天级习惯 / 永久记忆图。每层都有独立的运行频率与生物学合理的时间常数（第 5、6 章）。

5. **面向"活着"的可观测性。** 我们把整个皮层下系统的内在状态（膜电位、突触权重热图、调质浓度时间序列、习惯轨迹、做梦报告）暴露给可视化仪表盘，使得"看见一个数字生命的内心活动"第一次成为可能（第 11 章）。

我们对这些贡献保持必要的克制：项目尚处于早期阶段，多个机制（特别是 SNN 的长期学习效能与做梦巩固的实证收益）仍待大规模实测验证；同时，工程实现中存在已知的债务（详见第 13 章）。本报告的目标不是宣告胜利，而是**完整、严格、诚实**地记录一个把哲学命题逐步落地为运行系统的过程。
