# 第 2 章 · 背景与相关工作

> *"课程报告不需要把所有相邻领域都纳入大一统比较。真正重要的是说明：为什么这个项目适合作为智能感知与无人系统课程中的 Agent 系统实践。"*

---

本章只保留与 Neo-MoFox 课程定位直接相关的代表性工作。我们不再追求完整综述，也不把所有 AI 伙伴、认知架构、神经形态硬件和强化学习系统放进同一个大矩阵中比较。对本报告而言，更有用的问题是：

1. 现有 LLM Agent 为什么容易停留在“调用即存在”的离散范式？
2. 经典认知架构给持续运行和模块协作提供了哪些启发？
3. 记忆型 Agent 与神经启发系统分别解决了哪些局部问题？
4. Neo-MoFox 作为课程项目，如何把这些思想组织成一个可运行的自主系统闭环？

## 2.1 从 AI 伙伴到 Agent 无人系统

商业 AI 伙伴系统（例如 Replika、Character.AI、Pi）证明了大语言模型可以提供强交互体验，但它们通常仍是**会话触发型系统**：用户输入到来时，系统检索上下文并调用模型；输入结束后，系统大部分内部状态停止演化。用户感受到的“连续性”更多来自历史摘要、人格设定和 prompt 组织，而不是系统在物理时间中的持续运行。

这类系统的价值不需要否定。它们说明自然语言接口、长期用户档案和人格一致性确实重要。但从《智能感知与无人系统》的课程视角看，这还不足以构成自主系统。无人系统更关心的是：当外部输入中断、通信不稳定、环境缓慢漂移时，系统是否仍能维持状态估计、感知增益和决策倾向的连续更新。

Neo-MoFox 因此把“AI 伙伴”重新解释为一种**软件 Agent 形态的无人系统**：对话只是外部交互通道，真正的系统主体是后台运行的 Life Engine、事件流、记忆图、调质变量和 SNN 快层。这个转译让项目从产品叙事回到课程主题：持续感知、自主决策、在线学习和状态持久化。

## 2.2 经典认知架构的启发

SOAR、ACT-R、LIDA 等经典认知架构并不是本报告的主要对比对象，但它们给了 Neo-MoFox 三个重要启发。

第一，智能系统需要**长生命周期状态**。SOAR 的持续决策循环和 LIDA 的认知周期都强调系统不应只在外部查询到来时才“启动”。这一点对应 Neo-MoFox 的 Life Engine 心跳：系统以固定节律推进内部状态，而不是等待下一次 LLM 调用。

第二，智能不是单个模块的属性。经典认知架构通常把工作记忆、长期记忆、感知接口和行动选择分开建模，这与 Neo-MoFox 的三层结构相似：LLM 负责语言与高层推理，SNN 负责快速事件响应，调质 ODE 负责慢速情态调节，记忆图负责长期状态沉淀。

第三，学习可以发生在部署期。SOAR 的 chunking、ACT-R 的激活衰减和 LIDA 的多机制学习都说明，系统不必完全依赖离线训练。Neo-MoFox 的软 STDP、Hebbian 强化和习惯 streak 继承的是这种“在线适应”的工程思想，而不是试图复刻任何一个认知架构。

## 2.3 现代 LLM Agent 的价值与边界

AutoGPT、BabyAGI、Voyager、Generative Agents 等工作推动了 LLM Agent 的实践：模型可以调用工具、规划任务、检索记忆、生成行动序列，甚至在模拟环境中表现出社会行为。这些工作对 Neo-MoFox 很有价值，因为它们证明了 LLM 不只适合聊天，也可以成为系统中的决策与协调模块。

但它们多数仍以 **LLM 调用** 为主循环。系统的“自主性”常常来自任务列表、工具链和 prompt 驱动的反思，而不是来自一个低成本、持续运行的内部状态层。换句话说，它们解决的是“如何让模型做更多事”，而 Neo-MoFox 关心的是“当模型没有被调用时，系统还在不在演化”。

在无人系统语境下，这个差异很关键。无人车、无人机或长期部署的服务机器人不能只在高层规划器运行时才维护状态；它们需要底层状态估计、传感器融合、控制回路和健康监测持续运行。Neo-MoFox 把 SNN、调质、记忆衰减和心跳看作这种底层循环的 Agent 软件版本。

## 2.4 记忆型 Agent 与神经启发机制

MemGPT、Letta、Mem0、Zep 等记忆型 Agent 说明，长期记忆不能只靠上下文窗口堆叠，必须引入外部存储、检索和摘要机制。Neo-MoFox 接受这一点，但进一步强调：记忆不应只是被动仓库，还可以成为随时间衰减、被使用强化、在离线阶段重新组织的动态结构。

神经启发系统提供了另一类参考。脉冲神经网络、STDP、调质动力学和睡眠巩固都来自神经科学或神经形态计算传统。本报告不会声称 Neo-MoFox 在这些方向上达到了硬件或科研前沿水平。它的课程价值在于：把这些机制以简化、可运行、可观察的方式放进一个 LLM Agent 框架中，让“连续状态”和“在线适应”不再只是 prompt 里的描述。

这也是 Neo-MoFox 与单纯 RAG 记忆、单纯工具调用 Agent、单纯聊天产品的主要区别：它尝试把多个低层动态过程作为系统背景，而不是把所有智能行为都压在一次 LLM 推理上。

## 2.5 课程报告中的设计空间

为了避免过度比较，本报告采用三个足够解释课程项目的维度：

| 维度 | 关注问题 | Neo-MoFox 的实现 |
|---|---|---|
| 连续性 | 两次外部输入之间系统是否仍在演化 | Life Engine 心跳、状态持久化、记忆衰减 |
| 感知与状态 | 系统如何形成可更新的内部状态 | SNN 快层、调质 ODE、事件流 |
| 自主与学习 | 系统如何在部署期改变自身行为倾向 | 软 STDP、Hebbian 强化、习惯 streak、做梦巩固 |

## 2.6 小结

本章将相关工作收敛为四类代表性参照：AI 伙伴、经典认知架构、现代 LLM Agent、记忆型与神经启发系统。这样的组织更符合课程报告的目的：说明 Neo-MoFox 为什么适合被视为一个软件形态的 Agent 无人系统原型，而不是证明它在所有相邻领域中都全面领先。

下一章将把“连续性、自下而上学习、系统涌现”三条设计原则形式化，并说明它们如何对应智能感知与无人系统中的感知连续性、在线适应和系统级自主性。
